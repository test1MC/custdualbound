Overview

The release of GOCAD Mining Suite 17 update2 focuses on usability and stability with the introduction of Mining Categories and Classifications to the Assign Data to Geologic Features window. Additionally, we have ensured compatibility with new functionality in upcoming releases of other Mira Geoscience products, namely VPmg and Geoscience ANALYST.

There are new tools and fixes in the following modules:

  • Mining Utilities
  • Targeting Workflow
  • Potential Fields

 

New and updated functionality

Mining Utilities

  • To facilitate more efficient organization in the Object Display Lists, we have added the mining data categories to the GOCAD Assign Data to Geologic Features window .
    Categories included are:
    • Fault, Horizon, Cultural, Geochemical Data, Airborne Geophysical Data, Ground Geophysical Data, Geotechnical Data, Mining Infrastructure, Physical Rock Property Data, Structural Data, and Fractures.

    Full details of the functionality of this dialogue are available from the SKUA-GOCAD help, however, basic details including panel functionality are extracted below:

  1. Examine modelling objects that are not associated with features. All objects in the project that can be classified as Faults, Horizons, Intrusive Boundaries, Fluid Contacts, Cultural data or Formations appear in this list. (Wells and voxets are not shown.)
    In this panel you can filter each column, view plots and statistics for objects, export objects, and add objects to your object lists.
  2. Show or hide the list of features of the type you want to assign data to, where each icon represents a feature category.
  3. Work with features (of the type selected at the top of the window), assigned objects, and find information about features and objects.
  4. Create, edit, and delete features, assign or remove objects from features, and, for horizon features, access the Stratigraphic Column Editor.
  5. Expand the filtering area and use tools to filter the list of features and objects. For example, you can filter the features shown by those that contain objects assigned to a given modelling role.
  6. Assign or change the modelling role and feature classification for selected objects.

  • Register Image from Points (Voxet \ Edit) now copies all properties in the 2D Voxet rather than requiring you to select only one property.

Targeting Workflow

  • When using the Weights of Evidence technique, you can now select a user-defined region when selecting the Prediction Model Volumes at the start of the workflow. This allows you to take advantage of the method's ability to deal with missing information and generate targets even where data may not be present in the entire volume.

     

Potential Fields

  • When importing data from VPmg model files, GOCAD Mining Suite now performes a check between the data range of the data being imported and any other previously imported VPmg model data of the same type, e.g., GZ, GUV, TMI, among others. If the difference in the data ranges is greater than a total of 20%, a new property display template is created with an incremented name. If a property display template already exists within this limit, the system uses that property display template.
    This new behaviour helps VPmg users who are inverting different data sets with different background values in the same project (e.g., local and regional data sets).
  • All VPmg data imports now create property names directly from the data section header of the model file.
  • VPmg 9.2 will support models that are rotated from a N-S orientation. These can now be exported directly from rotated Voxet models within GOCAD Mining Suite; all other model export options currently display an error message when using a rotated Voxet to define model extents.
  • When creating control files to run VPmg inversions, you can incize local rotated models into regional models only if the rotation angle of the two models is the same. An error message will display if the rotation angles are different.
  • With the introduction of the support of rotated models in VPmg 9.2, GOCAD Mining Suite now supports the import of these models to both Grid objects and Surfaces.

Fixed issues

Mining Utilities

We have fixed a SKUA-GOCAD issue within GOCAD Mining Suite that occurs after saving a selection of objects into a new project and then opening that project. In this instance, a 3D Viewer was not immediately available to the user. This issue initially created problems with the ioGAS Link plugin.

SKUA-GOCAD™

SKUA-GOCAD 17 support packs 1, 2, 3, 4, and 5 have been included in our installer as they resolve issues reported by our clients, such as:

  • Viewing Voxet and 2D Survey objects with Viewer-only licences.
  • Creating cross sections from Voxet sections by right-clicking them in the 3D Viewer.
  • Preventing a complete rebuild of faults when updating the fault network after exiting and reopening the project in the Structure and Stratigraphy (SKUA) workflow.
  • Using a property script on well logs in a macro.
  • A model not being properly sealed when creating a Model3d from a SKUA Model (Surface commands \ Model3d \ From SKUA Model) with erosion and faults.
  • Editing the scale for a log in a cross section or well section view by accessing the Style editor for the log from the Well Track Templates browser.

Supported third-party programs and APIs

GOCAD Mining Suite 17 update1 supports the following software versions of our partners:

  • AcQuire API 4.5.6.3
  • Geosoft API 8.4*
  • PFD3D API 2.13.8
  • ioGAS 7.0 and earlier
  • UBC 5.0.1 and earlier
  • VPmg 9.2 and earlier
  • VPem1D 4.1 and earlier

*To use any of our commands that rely on the Geosoft API, Geosoft requires that you have a Geosoft ID. A Geosoft ID can be obtained directly from the Geosoft website: https://my.geosoft.com/createnew. If you have any Geosoft products, you likely already have a Geosoft ID.

Supported Operating Systems

GOCAD Mining Suite 17 update1 is supported on Microsoft Windows x64 Systems: Windows 7, Windows 8.1, and Windows 10.
