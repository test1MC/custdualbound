Defining your data

Once you have created your project, you must configure Geoscience INTEGRATOR so that it can accurately interpret the data that you will be importing.

A project contains several elements to define and categorize. Therefore, it is helpful to understand the hierarchy of the elements of a project:

themes

themes

themes define and categorize the nature of the data contained within a project. For example: data related to events associated with small movements beneath the Earth’s surface would fall under the Microseismic theme. A project must have one or more themes.

See the themes section for a breakdown of the themes available in Geoscience INTEGRATOR.

To select a theme for your data:

  1. In the Geoscience INTEGRATOR menu, click data set explorer (1).
  2. Click Current theme to access the drop-down list of themes.
  3. Select the appropriate theme for your data.
Note: Geoscience INTEGRATOR defaults to the most recently selected theme of the most recently opened project.

 

 data sets

data sets

data sets are groups of related information contained in a theme. When you import data into Geoscience INTEGRATOR you are creating a data set by assigning to the data various Project settings that help to organize and categorize the information.

 Project settings

Project settings

Project settings consist of several categories designed to organize and arrange data within a data set. In Geoscience INTEGRATOR, the Project settings are:

  • Units
  • File types
  • Classifications
  • Assigned users
  • Neighourhoods
  • Sample media
  • Shifts
Note: Under default permissions, only system administrators and project managers can make changes to Project settings.

 

 Units

Units

Geoscience INTEGRATOR requires that you define within it the units of measurement of your data before you import.

Geoscience INTEGRATOR contains a default list of common units when initially installed.

To define a data set’s units:

  1. In the Geoscience INTEGRATOR menu, click Project settings(i), then click Units(ii).



  2. In the Units table, click Add new unit.
  3. In the Add new unit dialog box, add the unit name abbreviation and its description.
  4. Click Add. Once you see the Unit successfully uploaded message click Done.
Note: We recommend creating a unit named “unitless” (or something similar) for properties that do not have a unit, such as seismic indexes.

 

 Classifications

Classifications

Classifications act as a container for subsets of data (known as classes) in a parent-child style of relationship. An example of a Classification and its classes for drillhole interval data, where Rock code is the classification, could be something like this:

  • Rock Code:
    • Granite
    • Norite
    • Ore

This is useful in that it prevents you from having to type each rock name into your template at import.

To define a data set’s classifications:

  1. In the Geoscience INTEGRATOR menu, click Project settings, then click Classifications.
  2. In the System classifications table, give a name and description to your classification.
  3. Click Add classification, and if necessary, define its classes, by:
    1. Clicking the Classification’s edit button (insert button)
    2. In the Class name field of the Classes for classification window, name the class.
    3. Click Add Class.
Note: If you plan to add numerous classes and wish to avoid typing them out by hand you can use the option to import the list of classes from a CSVfile. To do so, click Import from file in the Classes for classification window and follow the given instructions.
Note: It is possible to generate supplementary classifications while importing data (see Data Import section of this user guide). Classifications created at import will only apply to the data imported via their specific import template and will not be added to the list of general project classifications.

 

 Neighbourhoods

Neighbourhoods

Neighbourhoods are 3D search spaces within a project, allowing you to give names to spatial regions/volumes within a project. For example, a Neighbourhood in a mine could represent a level, a stope, a drive, or a shaft area, while in mineral exploration, a Neighbourhood could represent, a mining claim outline.

To create a Neighbourhood:

  1. Project settings, then click Neighbourhoods.
  2. In the Current neighbourhoods table, click the Create neighbourhood button.
  3. In the Create neighbourhood dialog, give your neighbourhood a name and a description.
  4. Click Add, then click Close.
Note: Make note about Importing a neighbourhood.

Neighbourhoods are composed of one or more 3D volumes with each volume being delimited by either an XYZ bounding box or a polygon with a defined Z-axis range.

  • Bounding box: A cuboid defined by its south-western and north-eastern coordinates.
  • Polygon: Comprised of an unlimited number of XY points that join to create a closed 2D polygon, where the last point joins with the first to complete the polygon. To turn the 2D polygon into a 3D volume, you must define maximum and minimum Z-axis values.
Note: Neighbourhoods are independent from one another; they can overlap. Therefore, a particular XYZ point can belong to many neighbourhoods.

 

 Shifts

Shifts

Shifts are set portions of time within a 24-hour period, allowing project managers to set active time slots for a project. Shifts facilitate time queries as project users will not need to enter specific start and end time values. Instead, they can simply select specific shift names.

We recommend that shifts do not overlap, however, Geoscience INTEGRATOR does not limit or control this. It is up to the project manager to decide how to set and use shifts. Project managers can edit shift names, descriptions, and Start time and End Time values. They can also delete shifts.

To set shifts in a project:

  1. Project settings, then click Shifts.
  2. In the Current shifts table, click Add new shift.
  3. In the Add new shift dialog:
    1. Give the shift a name and description.
    2. Define the start and end times.
  4. Click Add, then click Close.

Assigned users

Assigned users

Project managers or system administrators can manage the user roles within a specific project. It contains a list of user accounts for the project, including username, email address, organization, and assigned role. System administrators and project managers can remove user accounts from the selected project, change user roles within a project, or remove users from projects.

To add a new user to a project, select the user from the Assign user drop down menu, and select the appropriate role for the user from the second drop down menu. When finished, click Save.

Note: Users must already be in the system before you can assign a user role. To add a user to the system, go to System administration > Users, and complete the Add new user procedure.
