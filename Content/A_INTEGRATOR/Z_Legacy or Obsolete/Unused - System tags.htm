System tags

System tags are theme-based and assigned automatically at the system level upon importing data. For example, all data imported into the Microseismic theme will be assigned the Microseismic theme system tag. Geoscience INTEGRATOR provides a list of all system tags with data on the right-hand side of the Tags screen. You can click on any of the system tag theme names to view a list of all data sets associated with that theme.

System tags are particularly useful to further filter data that is tagged with a project or user tag, for a theme. It is possible to search through tagged data by combining a user or project tag and a system tag. For example, if we have many data types tagged with project tag ‘February 7th, 1999 level 755 rockburst’, it is possible to filter that tagged data to only view which of this data pertains to the Microseismic theme.

To view tagged data that pertains to a particular theme:

  1. From the Current tags table of the Tags page, Select one or more tags .
  2. Click the magnifying glass to see all the data associated to those tags.
  3. Click any of the system tags listed at right to see only the data associated to that system tag.



In general, system tags are created automatically in Geoscience INTEGRATOR, however, there are exceptions for some themes. These particular themes can support data of various nature as opposed to themes that support explicit (or particular) subjects. These ‘support themes’ are:

  • Drillholes (only drillhole interval and point logs templates)
  • Samples
  • Observation points

For example, you may have a CSV file that contains Drillhole interval data, where the properties listed are drillhole ID, from, to, gold, copper, density, resistivity, and RQD. All these properties are imported under the Drillholes theme because the data was collected in a drillhole and possesses drillhole ID, from and to fields. However, gold and copper , by their nature, are part of the Geochemistry & Mineralogy theme, while density and resistivity are part of the Physical properties theme, while RQD is part of the Geotechnical monitoring theme. In these particular themes, you can identify to which system tag the data belongs.

To apply cross-theme system tagging at import:

  1. In the Import CSV page, locate the template you will use to import data to apply a cross-theme system tag to.
  2. Select the user-created property you want to apply a cross-theme tag to.



  3. Select one or many system tags from the list.

  4. Click Save.
