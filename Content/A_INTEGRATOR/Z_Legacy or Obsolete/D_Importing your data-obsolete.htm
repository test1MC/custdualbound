Importing your data

Geoscience INTEGRATOR requires that you import your data to a template so it can organize the data in the most efficient and useful way possible. To do so, you must define the properties of a data set within a template of a theme.

Within each theme the templates differ slightly to accommodate varying properties associated with the theme. You must select and create your template(s) based on the characteristics of your data.

Templates consist of:

  • Mandatory properties: At minimum, this includes date, time, and the XYZ co-ordinates.
  • User created properties: Includes anything contained within the theme associated with your data set.
Note: The properties you create in the Geoscience INTEGRATOR template must correspond with the properties that define your data set.

To create a data import template:

  1. In the Geoscience INTEGRATOR menu, click Import(i), then click Data import(ii).
  2. Select the type of file you wish to import (CSV, ASCII, or Manual entry)
Note: Only certain themes allow ASCII import. For this tutorial, we will use CSV.
  1. When the template box(es) appear, choose the template in which your data belongs.
Note: Certain themes have a sub-category or several sub-categories. Choose the one that best fits your data.
  1. In the appropriate template box, click Create new.
  2. When the Create new (theme) template table appears, name your template and click Create.

 

Now that you have a template to work with, you must define the properties of your data.

To define mandatory properties:

  1. In the Properties table, under Mandatory properties, determine if the naming convention of the selected template corresponds exactly with the naming convention of the same properties in your data set. If so, move to the next step. If not:
    1. Click the property in the Name column that you wish to edit.
    2. In the Property: dialog, type the property name so it matches the naming convention of your data set in the Mapped to text field.
Note: Depending on the theme, some of the options fields in the Property: dialog will differ between themes.
  1. Click Update.

 

To define user created properties:

  1. In the Properties table, click the Add new property button.
  2. In the Add new property dialog box, add the name of the property in your data set you wish to add to the template.
  3. Define the property types (see appendix for Project settings of the property types).
  4. Determine if the property is required for……
  5. Define the unit in the drop-down menu (see the Units section if you have not yet set the units for the project).
  6. Determine if the property should have a no data value. If so, enter the value into Property no data value text box. If not, leave it blank.
  7. Click Save and repeat with any remaining properties.
Note: Alternatively, you can click the Read Headers button, which opens the windows explorer and allows you to select a CSV file. Once you select a file, Geoscience INTEGRATOR will scan the file and populate the User created properties table based on the headings in your data set CSV file.

 

Now that you have all the properties of your data set defined in your template, you can import the CSV file that contains the data set into Geoscience INTEGRATOR.

To import your CSV file/data set into a template in Geoscience INTEGRATOR:

  1. In the Geoscience INTEGRATOR menu, click Import > Data import > CSV file.
  2. Within the Import…data screen, navigate to and click the template you created containing the properties you defined from your data set.
  3. Click Upload data.
  4. Click data set and give your data set a name and description, then click Add data set. (You may wish to add a tag in the section just below this section by following the procedure.)
  5. Click Upload data file.
  6. Select the CSV file from your Windows browser and click Open.

Now that your data set is importing or has imported, you can check its status by clicking on Import > Data import > Batches. This page displays information associated with data imports.

Note: If you get an error message before the import has finished, it relates to a discrepancy between the naming convention of the data set properties between what you defined in the template versus how it’s written in your CSV file. However, if you get an error in the data, it will display in the Errors column in the Batches page. Here, you can click on the number in the cell, and see a description of the error.
