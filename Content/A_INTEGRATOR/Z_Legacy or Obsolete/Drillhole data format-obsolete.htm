Drillhole data format

This information is designed to provide you with the necessary formatting needed for drillhole data input in Geoscience INTEGRATOR.

The Drillhole theme in Geoscience INTEGRATOR can accommodate collar, deviation, interval, and distance-specific data. The accepted format is comma-separated values (.csv).

Drillhole collar data

This applies to general hole info like Hole ID, coordinates, length, dates, contractor, geologist, etc.

Format example:

Hole ID,XYZ,length,Drilling date,Drilling contractor,Logged by

DH001,10300,30200,300,123,2010-10-28,ABC Drilling inc.,J. Smith

DH002,10350,30175,290,234,2010-10-30,ABC Drilling inc.,B. Goode

HH321,13000,30200,299,321,2011-03-15,DEF Drillers ltd.,J. Smith

The red columns (Hole ID, 3D coordinates and hole length) are the minimal mandatory data needed.

Geoscience INTEGRATOR does not allow two holes to have the same name, so Hole ID values needs to be unique (per project).

 

Drillhole deviation data

This applies to tracing the hole in 3D, typically deviation survey data like depth, azimuth, dip, instrument type or name.

Format example:

Hole ID,distance,azimuth,dip,Instrument

DH001,0,180,-45,Instrument #1

DH001,25,178,-44,Instrument #1

DH001,50,176,-40,Instrument #1

DH002,0,180,-50,Instrument #2

DH002,30,177,-49,Instrument #2

The red columns (Hole ID, distance, azimuth, dip) are the minimal mandatory data needed.

The combination of Hole ID and distance needs to be unique, because Geoscience INTEGRATOR only accepts one set of measurement per distance down the hole per hole.

For proper 3D rendering, azimuth values should be between 0 and 360 and dips between -minus 90 and 90: negative dips creating downward trending drillholes, while positive dips create upward trending drillholes, such as in underground drilling.

 

Drillhole interval data

This applies to information related to various intervals of holes like rock types, alterations, structures, mineralizations, RQD, sample #, assay results, etc.

Format example:

Hole ID,From,To,Rock type,Alteration,RQD

DH001,0,6,Overburden,

DH001,6,63,Sandstone,CL1,90

DH001,63,102,Mudstone,CL2,75

DH001,102,123,Gabbro,CL1,95

DH002,0,9,Overburden,

DH002,9,57,Sandstone,CL2,85

DH002,57,252,Wacke,SR,90

 

Hole ID,From,To,Sample,Au,Ag

DH001,36,39,SP1111,45,12

DH001,39,42,SP1112,30,1

DH002,70,80,SP1234,100,12

The red columns (Hole ID, From, To) are the minimal mandatory data needed.

The combination of Hole ID, From and To needs to be unique, because Geoscience INTEGRATOR only accepts one set of values per interval per hole.

 

Drillhole distance-specific data

This applies to information related to a specific distance down the hole like spot test readings of magnetic susceptibility or handheld XRF analyzers)

Format example:

Hole ID,Distance,Mag sus.,Fe content (pXRF)

DH001,56.5,45678,10.7

DH001,100.2,60053,23.4

DH002,35,30002,6.4

DH002,92.3,40021,12.1

The red columns (Hole ID, Distance) are the minimal mandatory data needed.

The combination of Hole ID, Distance needs to be unique, because Geoscience INTEGRATOR only accepts one set of values per distance per hole.
