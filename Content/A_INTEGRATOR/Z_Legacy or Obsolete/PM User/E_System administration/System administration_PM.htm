System administration

This chapter page describes the functionality of at the system administration level. Here, system administrators and project managers will be able to create, delete, update, and edit the following elements of projects:

  • Users
  • Organizations
  • Projects
  • Metadata: data sets, Import templates, Groups.

Each of these modules are made up of tables that list crucial information and metadata associated with project elements.

Note: The Users, Organizations, and Projects modules function on the system level, not the project level.

To add any of the above elements to a project (with the exception of Import templates and Groups), click the Add new ... button at the top right of the table, and define all the fields provided by in the various dialog boxes.

Editing metadata

You can only edit a data set of the theme selected in a project while you are in the Edit metadata module. Additionally, you cannot delete a data set that contains no data.

Import templates

The table within Import templates displays a list of the import templates within the selected theme. The Is data? column indicates if the import template is in use or not (y or n). You must delete any data associated with an import template before you can delete the template.

Groups are only available in specific themes.
