Overview

Data fusion is the integration of data from multiple sources to obtain information that is more useful than what individual data sets can provide. In , data fusion is available in 3D, for sample data, and in 4D for mine model data.

Through this process, offers powerful tools to support the analysis of complex or difficult-to-view relationships within the data. The data fusion tool compiles the data from various sources, thus saving time and limiting the risk of error caused by performing the tasks required to bring this data together manually.

Fused data can become the input for machine learning algorithms, which requires that data is brought onto the same support, with the values collocated in space and time.

In Geoscience INTEGRATOR, data fusion contains two components:

  • Samples compilation: Compiles sample-related data from different data sets in different themes that are referencing the same sample ID. (See "Samples compilation" on page 1.)
  • Mine model compilation: Compiles numerous geohazard-related properties, on points of a mine model as of a single datetime reference. (See "Mine model compilation" on page 1.)

Samples compilation

Samples vary in nature and come in many forms. They are often analyzed using different tools and techniques, and for numerous goals. makes it possible to import and associate data related to samples in multiple data sets and themes, depending both on the nature of the sample's properties, and how these properties relate to a theme. The samples compilation provides a tool to view all data associated with a sample across multiple data sets and themes.

There are four themes for sample-related data:

  • Samples
  • Geochemistry & mineralogy
  • Rock properties
  • Drillholes (optional for interval data)

For example, a drill core sample could have:

  • The rock type imported in the Drillholes interval data along with relative location data (from and to position in a drillhole that has a known collar location and path data).
  • Density, porosity, resistivity, and magnetic susceptibility imported under the Rock properties theme.
  • Assay results for all the elements and minerals under the Geochemistry & mineralogy theme.

In the case of sample data, fusion is possible because each sample-related theme requires a sample ID. The samples compilation component allows you to query all the data referring to a sample ID regardless of theme or data set.

The samples compilation user interface consists of two sections:

  • "Details tab" on page 1.
  • "Chart | Table tab" on page 1.

Details tab

The Details tab compiles a summary per sample ID of all the properties (organized by data sets and themes) and all the tags and files associated to the sample ID, along with the sample medium and the available location data.

The Details tab allows you to perform the following actions:

  • Filter the samples displayed by typing a search term in the Sample name field (queries sample IDs only, not properties, tags, etc.). Or select the sample ID check box and click Filter.
  • View all the properties (organized by data sets and themes) for a sample ID. You can click on a data set to go directly to the Data set explorer where that specific data set will display. (See "Data set explorer" on page 1.)
  • View the Tags associated to the sample IDs. You can click a tag name to go directly to the tags page with that specific tag selected to view all the data sets associated with that tag. (See "Tags" on page 1.)
  • View the Files and maps associated to the sample ID regardless of the theme. You can click a file or map to download it.
  • View, and possibly input or update location information on the sample by clicking Sample ID. If no location data has been entered yet, you can provide x, y, z coordinates or, select a Drillhole and provide Interval from and Interval to values.
If a sample has a red X in the last column, it means that there are no data records associated with that sample in the database. The links to tags and files remain. The sample ID then acts as a placeholder. Clicking the X is a permanent delete of the sample record.

Chart | Table tab

The Chart | Table tab allows you to view the data from different data sets and theme together in a chart and a data table.

Here, you will find the same filtering options and chart and table settings options that are available in the Data set explorer (see "Data set explorer" on page 1). However, the properties from all the data sets and sample-related themes are available, allowing you to analyze them together and explore cross-data set or cross-theme relationships. For example, you can view a Scatter plot of "iron content (Fe)" from the Geochemistry & mineralogy theme versus "rock density" from the Rock properties theme and observe a direct correlation between the two. Since it is a scatter plot, you can use a property to modulate the radius and another one to modulate the colour and allow further correlation. You can also view the data, compiled per sample ID, in a table.

Mine model compilation

The Mine model compilation component brings all the mine model data onto the same support, with the values collocated in space and time. This data fusion capability is fundamental to geohazard assessment.

It consists of the following three tabs:

  • Overview
  • Mine level plans
  • Compiled table

The Underground theme (within Mine models) merges collocated data together. The default collocation distance is 1/100th of the distance unit, meaning that points closer together than 0.01 in distance unit will be merged onto a single point.

Overview tab

This tab displays a box for each Mine model Underground data set of the project with a summary of quantity of points, imported properties, and system-computed properties.

Clicking a box will expand it and reveal the detailed list of properties, which can be either of the following:

  • Imported properties: User-imported properties that the system does not update. These are typically fixed properties that are not expected to vary in time, such as a date of excavation.
  • System-computed properties: The result of various computations using other data or files from the project. You can schedule or manually trigger these. For further information, see "Geohazard assessment" on page 1.
The various geohazard indices are not listed here as Mine model properties.

Mine level plans tab

This tab provides plan views of the Mine model points, per Mine level.
You will find the list of Mine levels and their minimum and maximum elevations in the project settings. Only users with specific permission settings can edit this list.

To display a mine level plan:

  1. Expand the Filters panel on the right and select one data set.
  2. Select the specific Mine levels you want to view (one or many).
  3. Click Apply changes to view the data.

Click the Limits section to customize level plan extents. This will apply the x, y limits to all the selected mine levels.

You can select a property from the Radius drop-down menu to adjust the size of the points according to the property values. If you provide a value in the Threshold number box, the size of the point will be based on whether the value is below or above the threshold value. The point above the threshold will be bigger.

Use the Radius value buttons to set how you want the values to correlate to the radius size of the points.

You can select a property from the Colour drop-down menu to adjust the colour of the points according to the property values. There is an option to apply a log scale to the colour mapping.

The field As of this date is to identify the specific datetime you want to query and is set by default to the moment you access the feature.

If you do not provide a value for As of this date for an instant or time interval type property, the most recent will be used. If you do provide a value, the most recent values before that datetime will be used.

Use the Colour spectrum buttons to set the colour scale to increase from blue to red or vice versa.

Compiled table tab

This tab is a snapshot of all the points in a data set at a specific moment. It displays all the properties you select from the list for all the points of the data set.

To display a Compiled table:

  1. Expand the Filters panel on the right and select one data set.
  2. Select the specific columns you want to display.
  3. Define which date and time you want to compile the data for. (Additionally, you can restrict the query to certain Neighbourhoods and Mine levels.)
  4. Click Apply changes to view the data.

For each Instant data property, the Compiled table will return the value that either corresponds either exactly to the specified datetime, or the most recent one before it.

For each time Interval data property, the Compiled table will return the value if the specified datetime falls into an interval for which data exists.

Since properties from the Static data do not vary in time, the Compiled table always returns the same value for each point of the mine model, regardless of the specified datetime.

The field As of this date is to identify the specific moment you want to query and is set by default to the moment you access the feature.

When you apply Neighbourhoods or Mine levels to filter a query, these filters are applied first, then the system computes the most recent dates on the resulting points, and finally compiles data for these dates and points. These results may differ from the most recent data related to all the Mine model points of the data set.
