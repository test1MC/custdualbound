Overview

Geohazard assessment as a module supports the automated computation of both knowledge-driven and data-driven hazard assessments. This module uses rules based on multiple user-defined criteria, and updates hazard assessments on a defined schedule, or when triggered by a user. The server performs spatial computations (such as proximity to structures) and other 4D hazard-related computations (such as microseismic event density) without requiring any user intervention.

Geohazard assessment occurs in two stages:

  • Data fusion process on the mine model points to obtain up-to-date property values.
  • Evaluation of the geohazard rules to obtain the geohazard index for the mine model points, as well as the calculation of a confidence index.

Data fusion: Mine model compilation

To evaluate the geohazard risks on the mine infrastructure, the Geohazard assessment module uses the points of the mine model data sets as its basis for spatial collocation. Through the Mine model compilation, the data fusion process brings together all necessary data for the computation of different geohazard indices.

In the Mine model compilation’s Overview tab, you will see a box for each Mine model data set of the project with a summary of the quantity of points, imported properties, and system-computed properties.

  • Imported properties: User-imported properties that are not updated by the system. These are typically fixed properties not expected to vary in time, such as date of excavation. These properties do not generally need updating prior to evaluating the geohazard rules.
  • System-computed properties: These are the result of various system computations using other data and / or files from the project. For time-varying properties, it is important to ensure the availability of up-to-date values prior to evaluating the geohazard rules.
    You can use the Calculate now button to manually trigger system-computed properties, or a calculation schedule can be customized.

The results of the system-computed properties updates are imported as instant properties in the Mine model data set with the datetime of computation as the time stamp. Once the properties are up-to-date, evaluation of the geohazard rules can take place. You can hover over Last calculation date (in orange) to display the timestamp of the completed import for that calculation.

Geohazard control room

The Geohazard control room manages the calculation of geohazard indices on-demand or on an established schedule.

This section of the user interface displays a summary information box for each geohazard of a project. The boxes indicate the most recent and next calculation dates (if scheduled), as well as the names of the associated Mine model data sets. Also indicated is the status, which can be one of the following: ready, calculating, failed, and threshold exceeded.

You can expand a geohazard summary information box by clicking anywhere within it. This will display additional information such as the schedule for updating the geohazard indices, the threshold and prior probability values, and the complete list of rules for calculating the geohazard and the confidence indices. You can hover over Last calculation date (in orange) to display the timestamp of the completed import for that calculation.

Each geohazard rule has a number and a name to identify it. Below the name, you will see each rule’s conditions and its positive and negative weights, when applicable. The conditions refer to the mine model data set properties, which can be static, time interval, or time instant.

    Geohazard calculations

    The geohazard calculations can take different forms. In each case, the system evaluates a list of rules representing conditions on the mine model points of the selected data set.

    The rules are based on user-defined relationships (knowledge-driven) or on the analysis of prior correlations (data-driven methods). Each rule involves one or more properties of the mine model points, and represents a condition. Rules have a positive weight (W+), and can also have a negative weight (W-).

    When a rule only has a positive weight, the system applies the weight if the condition is met. For example:

    Distance_to_ore_contact < 15m W+ = 10

    For each point of the mine model data set, if the property named Distance_to_ore_contact is smaller than 15, a weight of 10 is applied.

    When a rule has a positive and a negative weight, the system applies a positive weight if the condition is met, and a negative weight if the condition is not met. For example:

    Distance_to_ore_contact < 15m W+ = 10 W- = -2

    For each point of the mine model data set, if the property named Distance_to_ore_contact is smaller than 15, the system applies a weight of 10, and, if the value is greater or equal to 15, a weight of -2.

    When a rule cannot be evaluated, the system applies no weights. In the above example, if no Distance_to_ore_contact value is available at a point of the mine model data set, no weight is applied for this rule on that point.

    When all rules have been evaluated for each point of the mine model data set, a hazard index is calculated. It can be as simple as the sum of the weights, or can be more complex, for example when using expert system data-driven rules.

    Weights-of-Evidence

    Weights-of-Evidence is a quantitative probabilistic method for combining data in support of a certain hypothesis. This data-driven method can be used for assessment of potential future hazard based on analysis of prior correlations.

    When rules are established with this method, the system uses the following calculations to obtain the hazard index:

    Geohazard index value

    A high hazard index value indicates that most or all of the criteria evaluated via the rules were met, and that all or most of the conditions are present that are believed to be leading to the occurrence of the hazard.

    To help bring your attention to the potentially dangerous zones, the user interface will clearly identify the points where the hazard index is greater than the threshold value. Detailed information is available in the user interface via the geohazard information box and the History of calculation page, and in the auto-generated geohazard alert report sent via email.

    Confidence index

    The system calculates a confidence index for each point. The confidence index gives you an idea of the reliability of the computed hazard index for each point of the mine model.

    The system stores geohazard indices with the datetime of computation as the timestamp on the mine model points as instant data properties.

    When the geohazard indices calculations are complete, the system sends one or more reports via email to a user-defined list of recipients. There are two types of geohazard reports: a routine report and an threshold alert report.

    The routine report confirms to the geotechnical engineers that the system performed the calculations, and that the geohazard index threshold has not been reached for any points of the selected mine model data set.

    The system sends a threshold alert report only if the hazard index is greater than the set threshold value for at least one point of the mine model.

    The geohazard threshold alert report contains:

    • Name of the geohazard along with datetime of the hazard index calculations.
    • Level plans of the level(s) that have points where hazard index is greater than the threshold value with the mine model points coloured by geohazard index value.
    • The x, y, z coordinates of the point with the highest index value along with the list of rules, conditions, and property values that led to the high hazard index.

    The geohazard reports, along the other standard reports, remain available in the system for download for a period of one month, via the Reporting/ Report history page.
