Overview

Tags are text-string identifiers you can assign to data and files in , giving you the ability to filter tagged data of a certain theme via the Data set explorer, or to search tagged data across various themes within a project via the tags page.

This is a powerful and flexible concept that will allow you to associate data that may look unrelated but can be meaningful to you.

An example would be to have a tag for all data and files related to a rockburst incident. The tag could be named "February 7th, 1999 level 755 rockburst." This tag could include microseismic, blasting, prior stress and ground deformation data records available in the area, incident reports, photos, and specific mine plans where damage has been identified, among others.

You can apply tags and search for data and files based on tags. You can apply the same tag to any data type, meaning that query by tag can result in the return of multiple types of data, files, and documents.

There are two types of tags in :

  • Project: Tags created by the project manager.
  • User: Tags created by users.

Project and User tags

  1. Clicking Tags in the menu opens the Tags page.
  2. Clicking next to Your user tags, Project tags, and Others' user tags, toggles on and off the display of these different types of tags.
  3. Clicking +Add new tag opens a dialogue box where you can create a new user or project tag depending on user permissions level of access.
  4. Clicking allows you to edit an existing tag properties (if permissions allow).
  5. Clicking deletes the tag.
  6. Clicking a number in the data sets column opens a dialogue box where you can view and associate data sets with the tag you are creating or editing.
  7. Clicking with a tag selected (using check box) allows you to view and explore all items associated to the selected tag.
    You can select multiple tags before clicking the magnifying glass to explore them. Then the results are restricted to the data sets and records that carry all of the selected tags.

Exploring tags

When you select a tag in the Current tags table of the Tags page, you can explore it by clicking the magnifying glass. Each theme that has one or more tags assigned to it will have its own information box, which you can open by clicking Show results.

The Data set name column lists the tagged data sets in the theme. You can click a data set name to navigate directly to it in the Data set explorer page. The Number of elements column functions as follows:

  • If the value is 0, only the data set is tagged.
  • If the value is ≥ 1, that represents the number of elements (records) tagged in the data set. Clicking on that number will expend the table, allowing you to see further details of the data. The details listed in these tables are theme-specific.

Creating and assigning tags

There are various ways to associate tags with data and files.

You can assign tags to individual data records, to a data set name (without associating the tag to the individual data records), to all the records of a data set, to files from the File manager, from the Maps / plans / sections or from the Documents library.

You can tag data either during import or on existing data already contained in the project.

Creating a new tag

  1. In the  menu, click Tags.
  2. In the Current tags table, click Add new tag.
  3. In the Add new tag dialogue box, give the tag a name, and an optional description.
  4. Decide if the tag should be a User or Project tag (permissions permitting) by selecting the appropriate button.
  5. Click Add, then click Close.

Assigning to a data set

  1. In the Current tags table, locate your tag, and click the number in the data sets column.
  2. In the tag's dialogue box, select one, many, or all of the data sets in the list, and click Save.

The number in the Data sets column represents the number of data sets assigned to a tag or that have at least one record associated to the tag.

Assigning to data records

  1. In the  menu, click Reporting, then click View data tables.
  2. On the right, locate your data set from Data set / Select a data set drop-down menus. (You may need to change the current theme.)
  3. Select any other filters you want to apply.
  4. Click Apply changes to populate the table with all the records that match your selections.
  5. In the generated records table, locate the record(s) you wish to tag, and click the number in the Tags column.
  6. In the Tags dialogue box, find the tag(s) you want to apply to the record(s), and click them.
  7. Click Save.
You can tag up to 1000 records by clicking Tag ALL results at the top right of the data table.
The granularity, or level of data, to which it is possible to associate a tag varies with each theme. For example, in the Ground deformation theme, only the stations / instruments can be tagged. It is not possible to tag a record of a specific measurement.

Assigning to all records during import

When importing a CSV file, you can decide to tag all the data records that are going to be imported from the file (if allowed by you user permissions).

  1. In the menu, click Import / CSV File.
  2. Select an import template.
  3. In the Upload data section, select one or many tags from the list.
This process tags each record being imported. Therefore, if you import a very large quantity of records, the tagging process may be time consuming, especially if many tags are being assigned.

Assigning to Files or Maps / plans / sections

  1. Click the number in the File manager and Maps / plans / sections in the Tags column of the files table.
  2. When the dialogue box appears, select a tag from the list and click Save.

Assigning to Documents

  1. In the Document library click the All documents tab, then the Edit button. From there, you can choose a document from the list, which will tag it.
  2. Click Update document.

Quick facts about Tags

  • Tagging records in a data set will make the data set show as tagged with the number of elements (records) when we view the tag in the tags page.
  • Tagging just a data set will show the data set as tagged with zero elements (records) when we view the tag in the tags page.
  • The ability to create / edit / delete project tags depends on user permissions defined by the system administrator.
  • Data and files do not have to be tagged.
  • Tagged data and files can have any number of tags.
  • There are no relationships between tags.
