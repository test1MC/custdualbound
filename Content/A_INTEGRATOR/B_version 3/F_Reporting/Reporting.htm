Overview

contains a reporting feature that you can configure to generate either on-demand, scheduled, or through data-triggering. It is also a tool you can use to analyze data.

You can dispatch custom-designed summary reports by email to designated recipients, or access them via the web browser interface.

System-designed geohazard reports are data-triggered. They automatically generate and send via email.

Reports will typically contain charts and tables combined with text elements.

The Reporting sections are described further below.

View data tables and charts

These two features function similarly to the Data set explorer. However, unlike the Data set explorer, they allow you to generate table-only and chart-only report items. (See "Data set explorer" on page 1.)

An important distinction is that in the View data tables and View data charts pages, it is possible to query data from any number of data sets together, instead of just one at a time.

The system will plot a maximum of 50,000 points on a single chart.

Assigning tags and files at the record level

From the View data tables module, you can associate or dissociate tags, files, and maps to specific records by clicking the number in the associated column, and selecting or deselecting the items from the list. Finish by clicking Save. You also have the option to tag or associate files / maps to all the results by clicking Tag ALL results or Files / maps in ALL results, which is not limited to the 300 rows displayed, but not more than 1000 at a time.

Reports

The Reports component of the Reporting tool applies to the custom-designed reports, not to the system-designed geohazard reports. It displays a table for custom-designed reports from which you can perform the following tasks:

  • View the report on current data, either in the browser (by clicking Preview) or in a downloaded PDF.
  • View individual items in the reports by clicking the number in the Items column.
  • View schedules, status, and recipients of each report.
  • Edit report settings, delete a report, and send reports to recipients.
  • Add new reports.

When generating a report, the system updates the data in the report with the data as it is at that moment. What is actually being saved as a report item is the query parameters, not the query results.
To keep the query results in their present state, click PDF to generate and download it to your computer. You can also upload it to the File manager to keep it inside the application.

To view a report:

  1. Locate the report you wish to view in the Name column.
  2. Click Preview to open the report in a separate browser window, or click PDF to generate and download the report.

Creating a report

  1. Click Add new report.

  2. Give a name (mandatory), and description (optional) to the report within the system.
  3. Determine if the report will be scheduled. If yes, see the scheduling parameters. If no, the parameters are hidden.
  4. For a scheduled report, set the schedule status:
    1. Active: The report is generated and sent via email at the defined frequency between the defined starting and ending datetimes. (You can also generate it on demand.)
    2. Inactive: You can prepare the report scheduling parameters but they will not be executed on the defined schedule until the status is changed to Active. (You can also generate it on demand.)
  5. Define the frequency of the distribution.
  6. Give the PDF file to be generated a name. The system will automatically add a datetime suffix.
  7. Create a distribution list of email recipients (comma separated). If no email address is provided, the system saves first the creator's email address.
  8. Click Save.

Once you complete these steps, the report will appear in the Reports table.

When you create charts and tables in the Data set explorer and reporting modules, you can add them to reports directly from those sections of the user interface. Since the system generates the chart and table separately when using this section, you can add just a table item or just a chart item to reports from here, as opposed to the Data set explorer, which always generates both together.

To add chart and table items to reports:

  1. Once you have built a chart and / or table from the Data set explorer or the Reporting page and clicked Apply changes, click Add report item+.
  2. In the Add a new report item dialogue box, select the report name from the drop-down list to add your chart and / or table to, and give a name to the report item you have just created.
  3. click Add, then click Close.

There is no limitation on how many items you can add to a report, however, only the first 100 lines of each table will appear in the report.

Formatting a report

On the Reports page, locate the report you want to edit, then click the number in the Items column to see the list of report items, or to add text items.

In the report's dialogue box, by default, the items will be listed in the order they were added. From there you can do the following:

  • Add text to the report (titles, sub-titles, paragraphs).
  • Delete items.
  • Click and drag items in the list to reorder. Finish by clicking Save order.

Report history

The Report history page lists the standard custom-designed and the geohazard system-designed reports that were sent via email, and sorted by sent date with the most recent at the top. It displays the Report title, Recipients, Type, Date sent, and Available until date.

PDF files from reports remain available for download by clicking on the report title for a period of one month from the time it was sent. The Available until column provides the date of that one-month mark after which the file will become unavailable for download and deleted from the system. If a report fails to send, you can still find it and download it from here.
