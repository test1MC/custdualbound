Connecting from

The application offers a powerful 3D visualization environment with a query interface that allows you to analyze data and files in a spatial context.

Once installed on your computer, connects to over a private network or the Internet.

To access your data from :

  1. Launch , and:
    1. Click the Geoscience INTEGRATOR Query tab, then
    2. Click the Connect now button. Alternatively, you can connect through the file menu by clicking Connect to Geoscience INTEGRATOR. (The latter method allows you to connect to a different project without exiting and re-entering the application.)
  2. In the Geoscience INTEGRATOR credentials dialogue box:
    1. Enter your email, password, and URL of a valid server
    2. Click OK
    3. Select a project from the drop-down list
    4. Click LOAD.

Once connected to a project, you will see the project summary information displayed in the data table in the workspace tab (below the viewport). This includes the project’s name and description, the x, y, z extents of the data, and a list of the files uploaded via the web browser user interface.

At the left of the user interface, you will see that the objects panel contains the themes for which there are data sets in the project. Expanding these displays the list of data sets in each theme.

Items listed in black in the Objects panel are data sets and objects that have imported data and can be queried, and are associated with Data sets (tables) in the Data set explorer. Greyed out data set names belong to the Data sets (files) section and they cannot be queried at the record level.

Clicking on a data set name in that list will load the Data set summary information and display it in the data table in a tab next to the Workspace tab. Although formatted differently, this is the same information that you see in the Data set explorer of ’s user interface, including the list of files associated to the data set.

The Geoscience INTEGRATOR query panel displays data set-specific filtering tools, while a yellow rectangular polygon displays in the viewport indicating the data extents.

As is the case with the Data set explorer, the filtering tools offered here are theme-specific. You can select the ones you would like to use by toggling on the appropriate boxes.

Query parameters

The Query parameters appear in the Geoscience INTEGRATOR Query panel, at the right of the  user interface. These are a series of filters as well as a tool to import certain objects and information. You can toggle on and off each one using the check box next to a parameter's name. Once selected, clicking Run at the bottom of the panel will filter and import your selections.

Bounding box

The Bounding Box query will filter the data based on X, Y and Z limits. By default, the from and to parameters are set to the extents of the entire data set. This option is available on all data themes.

To modify the bounding box extents:

  1. Digitize new from and to locations by either clicking on the corresponding target icon to the right, entering new values in the text boxes, or using the up and down arrows.
  2. Click Reset to max to revert to the entire X, Y, and Z extents.

To generate a bounding box that covers the area where most data is concentrated, click 2 standard deviations.

You can visualize the bounding box of this area of concentration clicking Show selection bounding box, the icon to the right.

You can drag a line in the viewport to nominate the X, Y, and Z extents. We recommend setting a view from top of the bounding box to draw the X and Y extents, and from a side view when drawing the Z extents.

Date

The Date option filters time-based data of the event-type themes. You can select dates using the calendars or enter them in the text boxes.

Another version of this filter is available to query the themes about objects with time-related data. Since these themes can hold many instant data for each object location and can only show one property value at a time per location, you need to provide a datetime value for it to select the instant and interval properties as of a specific datetime. The system will return the values of the closest instant record before that date and the interval data record that includes that date (if there is one).

Object

The name of object type varies from theme to theme.

This option refers to samples and other specific types of objects that you can use as filters in the Data set explorer.

Property Filters

This Property Filters option filters by property values. Press + to add another property filter.

The nature of the data in a theme will determine the query behaviour and options available in the property filter.
  • Text data: Is is the only option available. You must enter the entire character string in the text box.
  • Classification and binary: Is is the only option available. Expand using the down arrow and select one or more classes to import.
  • Float, integer, and date: Select one of the available operations and enter a value in the text box.

For Shifts, Properties, Tags, and Groups. simply toggle on using the check box and select among the available items in the project to import them.

When you have set all the desired filters, you can click Run at the bottom of the query panel to send the query. will only load the pertinent records that fit all the query parameters.

Viewing data import results

stores the data import results on special object types, depending on the data type. For example, Microseismic events are a special points object that stores the data on nodes referred to as Events.

For data sets that have groups (e.g., Earth models / 3D points sets), the query results are split into one object per group name under a timestamped folder. Similarly, drillholes are objects located within a timestamped folder, within the drillholes data set group. All other queries will be objects named according to a timestamp directly under the data set folder.

The viewport displays the imported data in 3D, and the data table displays a summary.

Viewing project files, maps, and documents

You can find project files, maps, and documents in the Workspace tab of the data table, where you can use a search bar to locate files of interest. Clicking on a file name will display a selection of options:

Import: Imports the file under the project or data set name in the objects panel. The data file has to be in a format supported by , for example GOCAD surface files, DWG and DXF files.

Open: The system will try to open the file using the program associated to that file type on the computer. For example, a photo file saved with the extension ..jpg will open in an image viewer application.

Save as: Downloads the file to your computer.

Pull: Downloads the file in the workspace and becomes available offline.

Cancel: Cancels the file selection.

Visualizing the data in 3D

Once you have imported your data into , you can take advantage of the numerous visualization tools to analyze it. You can view integrated data and models from multiple data sets and files together and analyze, interpret and share with others.

Before exiting , you can save the imported data using the File menu option. Doing so will create a file with a (.geoh5 extension) on your computer, which will remain available even after terminating the connection to .

It is only possible to connect to one project at a time.
