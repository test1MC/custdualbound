Data set explorer

The Data set explorer is the starting point to view the data sets available in a theme, allowing you to access description of them and access their data. It provides the list of data set names, along with their descriptions and theme-specific summary information to help you select those of interest to you. It also gives you access to numerous resources and tools designed to help you analyze the data and create report items such as charts and tables.

You can access it by clicking Data set explorer in the menu at left. It lists all the data sets available in the selected theme, split between two sections: Data sets (tables), and Data sets (files), depending on whether data has been imported in tables or only as linked files.

The main difference between storing data in tables or as files is that you can query the data sets listed under Data sets (tables) to generate charts and tables for analysis and reporting. Data sets listed under Data sets (files) provides links to, and descriptions of, data sets stored as files, in order to organize the pertinent files in the theme structure. You cannot query their data directly in , but you can download the files to use with other applications, such as or other specialized software.

In a situation where you might be expecting data in the future, you can create the data set ahead of time and it would stay under Data sets (files) as a placeholder prior to importing the data. After the first import, the data set will automatically be moved to the Data sets (tables) section.

For each data set, after its name and description, you will find a summary of general and theme-specific information. Neighbourhoods, Tags, Files / maps / documents, Properties, Data set range, and Metadata are always present in data set summaries. Depending on the active theme, the data set summaries may contain other columns such as Samples or Stations. This is simply a way to provide you with more information about the data set and is contextual to the type of data.

There are tooltips on all items so you can have a detailed preview of the content.

You can click anywhere on a data set box to expand it and view the detailed content summary.

Neighbourhoods

For data sets under Data sets (tables), neighbourhoods listed indicate that at least one record resides within them. For data sets under Data sets (files), it indicates that the volume specified by their x, y, z range overlaps the listed neighbourhoods.

When there are no neighbourhoods listed, it does not imply that no data records exist; it means that none of the data records from the data set reside in the project’s defined neighbourhoods, or that no defined neighbourhoods overlap the provided x, y, z ranges.

Establishing which neighbourhoods intersect a data set is done automatically by the database. Updating on import of new data or definition of new neighbourhoods is automatic.

Neighbourhoods are defined from the Project settings menu. To read more about them, see "Neighbourhoods" on page 1.

Tags

The list of tags for a data set indicates all the tags associated to the data set or to at least one record of the data set. Tags are text-string identifiers you can assign to data in .

The magnifying glass next to a tag from that list will bring you to the Tags page to see all the data also assigned to that tag.

To read more about tags, see "Tags" on page 1.

Files / maps / documents

The list of Files / maps / documents on a data set indicates all the files, maps / plans / sections and documents associated with the data set, or to at least one record of the data set. The strength of this feature lies in the fact that not only does import and store data values, but it can link any number of files to the data set or to individual data records.

Properties

The list of properties displays all the properties, and their units, for which data is available in the data set. In the Data sets (tables) section, the list is automatically updated as data is imported.

In the Data sets (files) section, the list can be manually entered by any user with proper permission level. This can be useful to document the properties that are available in an attached data file, to inform users without them needing to download the data file blindly and open it to see what it contains.

With proper permission level, you will see icons to manually populate that list.

Data set range

This box provides the minimum and maximum values for each 3D axis for the data in the data set. For time-related themes, it also provides the minimum and maximum values of the mandatory datetime field. In the Data sets (tables) section, the values are automatically updated as data is imported.

In the Data sets (files) section, just as for Properties (see previous paragraph), and for the same reason, the values can be manually entered by any user with proper permission level, using the same icons.

Manually providing the 3D axis ranges for a data set under the Data sets (files) section will allow the system to identify corresponding neighbourhoods in the Neighbourhoods box.

Metadata

The Metadata count indicates how many metadata fields have been associated to the data set, regardless of if they are populated or not. When the data set is expanded, each metadata field has its own box displaying their values.

With the required permission level, you can add or remove metadata fields associated with data sets as well as populate them.

Icon Functionality

This icon at the top right of a data set allows you to add Metadata fields to it.
Click this icon in the dotted box at the bottom of the data set summary to add a new field, and select one from the drop down list. You can add one of each and can populate them right away.
Click this icon at the top right to save your new metadata fields.
Click this icon at the top right to cancel your new metadata fields (i.e. exit without saving).
On existing metadata boxes, this icon allows you to edit their values.
On existing metadata boxes, this icon allows you to remove that metadata altogether from this data set.

The list of Metadata fields is defined through the Data set metadata page under the Project settings menu. (See "Data set metadata" on page 1.)
