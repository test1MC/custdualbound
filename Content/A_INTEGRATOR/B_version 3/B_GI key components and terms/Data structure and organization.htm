Projects summary

When you log into through the web browser application, the homepage you see is the Projects summary page, which displays an overview of the active project and any other projects you have permission to see. You can return to this page at any time by clicking on the Geoscience INTEGRATOR icon at the top left of your screen.

At the bottom right of the user interface, you can send a message to support by clicking on Contact support, or switch language preference by hovering over the language menus.

Projects

Projects are the highest-level item in the data structure of , and they typically represent mine sites or exploration properties. There is no restriction on what a project can be: it is a collection of data sets with a common coordinate system and files that have been grouped together under a single project name. User access is set per project.

The Project summary homepage provides an overview of general information pertaining to the projects:

  • the project name
  • the project description
  • the project manager
  • the project creator
  • the date of creation
  • the list of themes that currently contain data sets. Clicking on a theme from any project navigates you directly to the Data set explorer page of that project and theme, ready to access the data. (Themes and data sets are explained next.)

You can switch projects by selecting a different project from the All projects table or from any page at any time by hovering over the Project menu at the top of the page and selecting from the drop-down list.



Themes

Themes are containers for data sets, designed to organize the data based on its type. Having data grouped in themes helps you to locate data intuitively. (See also "Data sets" on page 1.)

Some themes are grouped together under a more general name. For example, you would find extensometer data in the Ground deformation theme, which is grouped under Geotechnical monitoring along with the Microseismic and Stress themes.

When launched again, defaults to the most recently selected theme of the most recently selected project.

Data sets

Data sets are useful containers for organizing the same sort of data into different subsets that are meaningful to users. For example, it could be helpful to separate surface drillholes and underground drillholes in the Drillholes theme, or data in the Samples theme could be split between outcrop samples and drill core samples, or water samples and soil samples. Data sets in are meant to have the same meaning by which geoscientists and engineers intuitively refer to "data sets."

Your data can be either imported into a new or already-existing Data set or uploaded as a file and linked to a Data set. In the first case, the data is stored as records in the database, which are combinations of mandatory properties and optional user-created properties that can be queried later for data analysis and reporting. In the second case, the linked data file can be downloaded at any time to open in any pertinent application. This way, any set of data can be stored and organized into the theme structure, regardless of format.
