Overview

This chapter describes how to import data to be stored as records into the database. See "File and document management" on page 1, for information on importing files, maps, and documents.

The standard way to import data in is by providing the data in CSV files and then running them through customized import templates to properly organize the data in the database. You can do this through the user interface or by using a customized monitoring folder system.

Additionally, certain themes provide a direct import option from specific types of files (GOCAD files) or manual data entry. Customized connectors can also allow you to pull data from other site databases.

Importing data using a CSV file

The import tool uses files in CSV format with the first row only containing the property names.

You must create Import templates for the application to know what data to import from the CSV files, and where and how to import it in the database. Each theme has its own relevant template options.

Using the import template system is a useful way to allow you to import any type of data, as data does not need to correspond to any fixed or predefined type.

Preparing import templates

  1. In the Theme list at the top the user interface, select the theme that best fits the data you want to import.
  2. In the menu, click Import/Data import/CSV file.
Depending on the theme, the template options differ slightly to accommodate their specificity (or data structure). You must select and create your template(s) based on the characteristics of your data. For that reason, depending on the theme, between one to five template categories are available.
  1. Choose the template category (Drillhole, Drillhole path, etc., as seen in the example below) to which your data belongs. From there you can create a new template or select an existing one to use. Click Create new if making a new one.
  2. To create a new template, name your template and click Create.

Mandatory and user-created properties

In a newly created template, you must define the properties of the data (columns of the CSV file) you plan to import as either mandatory or user-created.

Mandatory properties

The mandatory properties are the minimum expected properties to make a valid record for the given theme. For example, a theme for importing points (e.g. Observation points) must have all three spatial coordinates for the point to be properly located in space. A theme for documenting events (e.g. Incidents) requires 3D coordinates and a datetime. For this reason, the system will not permit importing records (rows of the CSV file) to a theme with missing mandatory properties.

Mandatory property names are fixed in the system. If these columns are named differently in your CSV file, you can either rename them in your file or you can indicate in the template the name of the column in your CSV file where the data resides. This is referred to as 'mapping' your properties.

To map mandatory properties from your CSV file:

  1. Click the property in the Name column that you want to map to a different name in your CSV.
  2. In the Property: name dialogue, replace the text in the Mapped to field to match the name of the column in your CSV file.
  3. Click Update.
Data types and units of mandatory properties are fixed.

User-created properties

The user-created properties are for the other, non-mandatory data you want to import. For example, in a Drillhole path template, in addition to the mandatory drillhole ID, distance, azimuth and dip, you might want to document the instrument type, or extra information the instrument might have recorded (roll, temperature, etc.).

To define a user-created property:

  1. Click Add new property in the Properties table.
  2. In the Add new property dialogue box, add the name of the property from your CSV file you plan to add to the template (it is case-insensitive). Criteria for a valid property name:
    1. Can combine letters, numbers and symbols, except for the apostrophe (').
    2. Maximum of 50 characters.
    3. Cannot be a name that is defined as mandatory on any theme, such as x, y, z, sample ID, azimuth,etc. See the full list under the Mandatory CSV properties column in "Theme template guide" on page 1.
    4. Needs to be unique per theme. For themes that offer more than one template category (except for Samples), you cannot use the same name in more than one category, but you can use it in any template of that category, provided that you set the same data type and same unit. (There is no limitation to using the same name for properties in different themes, regardless of type and unit.)
      • Exception: For all three template categories of the Samples theme, you can use the same property names (as long as they have the same types and units). Also, if you use the same name in more than one samples-related template category (that is all three Samples template categories plus Geochemistry and Mineralogy, Rock properties and Drillhole interval), it needs to have the same type and unit.
The system records the names you enter here in the database and uses them for identifying the data everywhere in the user interface and in reports. We recommend that you choose property names carefully because once data is imported under a property name, it cannot be renamed unless you remove all data of that property, modify the name in the template AND in your CSV file(s) and re-import your data.
  1. Define its property type from the six available types:

Property type Definition
Alphanumeric For free-form strings of text and / or numbers.

Note that if there are commas in these strings, you must put double-quotes (“ and “) at the beginning and the end for the importer not to confuse these commas with the ones separating the values in the CSV files.
Binary For binary data, such as on / off, ore / host, true / false.

Note that when selecting this option, you will be asked to enter both expected values.
Classifications
  • Classifications can be preset at the project level or defined as you are preparing the import template.
  • Selecting a project classification will display the list of classes.
  • It is possible to add new ones to this list, but these additions will only apply to the data you are preparing to import. The project classification will not be updated with the new classes.
  • If you do not select a project classification, simply enter the classes separated by a comma.
  • You will also be asked to identify whether or not the classes are case sensitive; if so, the values of your CSV must match exactly the capital and lowercase letters of the list of classes.
Datetime
  • You must indicate the date format the CSV file uses.
  • Your data can include time in either hh:mm:ss or just hh:mm format. This does not need to be set anywhere.
  • Note that if your data does not include time values, the system assumes 00:00:00 by default.
  • Indicate whether or not to allow the import of data where the date is in the future (greater than the current system date).
Float For numeric values with decimals.
Integer For numeric values without decimals.

  1. Determine if the property is required. If yes, the data to import using this template is required to hold values for that property for all the importing records (rows of the CSV file), or else the import will fail and report an error. This setting is useful to identify missing data. If you set the required field to no and there are empty entries for that property in the CSV file, the system will not halt the import process. Blanks or empty records are identified as no-data value (NDV) in the database.
  2. Select the unit from the drop-down menu. The list of Units is defined in the Project settings (see "Units" on page 1). You must have the required user permissions access to modify this list.
  3. Provide a Property no-data value. Some software or device output files hold a predefined string of characters for fields with no data instead of leaving them blank. If this is the case, enter the appropriate no-data value for the system to recognize it as such (e.g., -99999 per GOCAD standard).
  4. Click Save and repeat to add as many properties as needed.
There is no limit to the amount of user-created properties you can add.

The system provides the following settings under certain themes or data types:

  • Property under detection limit: Only offered on the Geochemistry & mineralogy templates. It can accommodate an "under detection limit" (UDL) per property. You should find these values on your lab certificates. Use the Format field to enter the exact string used in your CSV file and the Value field to enter that value. Example: Format: <0.01, Value: 0.01.
  • QA/QC rules: For quality control of float, integer or datetime type properties, you can set a minimum and a maximum value for validation of the data. If any value fails validation, the import process will fail and report an error.

Alternatively, you can click Read headers, which allows you to select a CSV file you plan on importing with this template. will scan the file and create the user-created property names based on the headers in your CSV file. You will still need to access the properties settings one by one to set their proper data type, unit, and other settings. Remember to click the Save button before switching to another property.

It is possible to clone a template, which you may want to do as a way to save time when importing data that use the same or similar property configurations. Follow these steps to do so:

  1. In the Clone import template dialogue box, give your new template a name.
  2. Define the Optional fields, which will display according to the theme you are in.
  3. Click Clone import template. You will now be automatically inside the newly cloned template, where you can edit settings as per your needs.

Importing data

From any newly created or existing template, you can import the data from your CSV file by assigning it to a data set. Themes that need group names such as 3D point sets, 3D regular grids, and Observation points will require that you provide one.

User permissions settings determine the ability to import data.

It is possible at this step to associate tags and files to the records you are about to import by selecting them in Tags and Files / Maps.

At the bottom, click Select file to select your CSV file, then Upload to trigger the import. Scroll to the top to see the confirmation message with a direct link to the Import history page to follow import status.


A template does not need user-created properties. As long as you have the mandatory properties, you can import your file.
You can re-use templates to import new records with the same properties as often as needed.

Specificity per theme type

Depending on the theme and template, you will have several actions available when importing data. The following tables list the available actions per template. Refer to the Uniqueness requirements to see the combination of properties that must be unique in order to import properly, as the system does not accept multiple values for a property (per record).

In the case of all points-type themes (except Microseismic sensors), the Group property exists to differentiate between data measured at the same location, but at different times. For example: If you use Observation points for structural measurements that are collected at the same location, such as fold axis and axial plane, or multiple phases of deformation, you could import them separately and assign them each to a pertinent group name.

Group names must be unique per data set, but can be reused in another data set.

As summarized in the table above, when you are ready to import new data through a template of a sample-related theme, you need to assign it a data set. It can be either an existing one or you can create a new one from here provided you have the required user permissions access.

The Uniqueness rules validate the data at import. The particularity of the sample-related themes is that the Uniqueness rules are not restricted to the theme but instead apply across all sample-related themes. This means that, even though you can reference a sample ID in multiple data sets across the sample-related themes, a sample ID can only have one value for a property name across those themes and data sets. For example: If a sample called "Samp1" already has a value for a property called "Density" under Rock properties, the system will not accept a value called "Density" under Samples for that "Samp1". This is to ensure that the Sample compilation module does not bring up conflicting or duplicated data.

The particular data structure of the sample-related themes is meant to allow you to import data from various specialities separately while still being linked together by their sample ID.

For example, a geologist could import the sample ID with its coordinates, date of sampling, and name of the person who took the sample in the Samples theme. Then the results for resistivity, magnetic susceptibility, and specific gravity could be imported subsequently in the Rock properties theme. Finally, when the assay results are back from the lab, they can be imported in the Geochemistry & mineralogy theme. The Samples compilation page allows you to see together on one line all the properties related to each sample ID, all samples-related themes included.

Samples with a location measured within a drillhole can be added to the Samples Compilation through the drillhole interval template. On the property settings, check Yes This property is a sample ID at the bottom of the box. All properties imported at the same time as this property will be sample-related.

You cannot assign a data set, tags, or files to the data imported through these four templates. As summarized in the table above, time-related themes generally have four templates to import data of varying structure. Each has a first template to import the basic mandatory data for each object type, which represents the instrument or piece of equipment: stress cells for the Stress theme, units for the Mobile equipment theme, stations for the remaining themes.

You must assign a data set for the objects you import, which can either be an existing one or a new one that you can create from this template, provided you have the required user permissions access. You can also associate tags and files to each record, as needed. As per the Uniqueness rule, only the specific object ID needs to be unique (per project and per theme, but across data sets). For example, a station ID "STA-01" cannot be found in two different data sets of the same theme, but "STA-01" and "STA-02" can have the same location.

After importing at least one batch of objects (station / unit / stress cell depending on the theme) with the theme's first template category, you have the option to use:

  • The theme’s Instant data template for properties that are measured at a specific datetime (e.g., instrument readings). A regular time-series is an example of what we refer to as instant data: data are recorded at a set of specific instants in time. Instant data is a generalization of a regular time-series, in which the measurement times do not have to be at a regular interval and the properties may be non-numeric.
  • The theme’s Interval data template for properties pertaining to an interval of time delimited by datetimes from and to (status, average of readings, etc.). Interval data is an extension of instant data, in which each measurement has an associated time interval (a start and stop time), as opposed to being associated with a single instant in time.
  • The theme’s Static data template for any non-time dependent properties related to the object in general (manufacturer, type, installation date, etc.).

You cannot assign a data set, tags, or files to the data imported through these three templates. They each refer to an object ID, which as an object is already organized into data sets and can already have tags and files linked. Refer to the Uniqueness rules for the combinations of properties that need to be unique for data to import successfully.

As highlighted in the "Theme template guide" on page 1, the Mobile equipment unit template does not include x, y, z properties, unlike the other themes of similar type (objects with time-related data). This is because the position of the mobile units can also vary in time, like other instant data properties. Therefore, this theme has two instant data templates: one with location and the other without. You can use the latter if you only want to record fluid levels, for example, and not the position of the units.

As summarized in the table above, there are four templates available to import data of varying structure in the Underground theme. Although its structure is similar to the other themes for objects with time-related data, its validation rules are different.

For the Underground theme, the object is points. Use the Underground points template to import the Underground points coordinates. You must assign a data set for the points you want to import, which can either be an existing one or a new one that you can create from this template, provided you have the required user permissions access. You can also associate tags and files to the data set, as needed.

As per the Uniqueness rules, the combination of data set and x, y, z needs to be unique, which means that you can import a point (single combination of x, y, z) in different data sets of the same theme (unlike the other objects with time-related data themes that only allow an object to exist in one data set).

In addition, for this theme, a collocation limit applies. A data set cannot contain points closer than 1/100th of the distance unit from another point. Such collocated points get their properties merged into one point. If the import of a collocated point attempts to import a previously populated value for a property, the system only keeps the first one, and ignores the other ones.

After importing at least one batch of points, you have the option to use:

  • The Underground instant data template for properties documented at a specific datetime (e.g., stress values).
  • The Underground interval data template for properties pertaining to an interval of time delimited by a datetime from and to (e.g., Type of ground support).
  • The Underground static data template for any non-time dependent properties related to the points in general (e.g., Excavation date, Rock code, etc.).

The same Underground points can be found in more than one data set. Therefore, in order for the system to reference the proper points, it is necessary to assign a data set when importing instant, interval or static data in this theme. Assigning tags and files is not an option when importing through these three templates because it is a feature for the object level only (Underground points).

Refer to the "Uniqueness rule", in table on page 1, for the combination of properties that need to be unique for data to import successfully through any of these templates.

Unlike other themes for objects with time-related data, the Mine models / Underground theme automatically creates points when importing instant, interval or static data, if they were not existing yet for the related x,y,z locations. To make it possible, you must assign a data set when using an instant, interval or static template. The points created this way must still abide by the collocation limitation.

You can import triangulated surface data to build charts and tables for data analysis purposes, or you can simply upload the files to the File manager or Maps / plans / sections to store and reference them to the project or to one or many data sets.

All the templates for 3D triangulated surfaces require that you assign a data set and a group, and allow the association of tags and files. The triangles templates, however, do not allow the creation of new data sets and groups: the triangle data has to reference already existing points. You must specify the group for the triangles to reference the correct set of points, since a same point location may exist in different groups.

Group names must be unique per data set, but can be reused in another data set.

Importing GOCAD files

The themes that hold 3D surfaces data (typically made up of points and triangles) have an option to import directly from the 3D GOCAD file. Only GOCAD ASCII files for triangulated surface (.ts) are currently supported.

  1. Under Data import, click GOCAD file.
  2. Click Select file, and locate in the explorer your .ts file.
  3. Assign a data set, a group and tags, as well as the all the properties found in the file. For the mandatory properties, you can select units and a display names. For the other user-created properties, you can select data types, units and NDV just like for the templates.
  4. Click Upload file.

Manual data entry

Manual data entry allows you to import a single record at a time, with no need for a CSV file. It is handy when you want to import only a small number of records. The system only supports the manual data entry for the Incidents theme at this time.

To enter data manually:

  1. Under Data import, click Manual data entry.
  2. Select a theme and then an existing template of that theme in the same project. You will see the fields where you can provide values in order to process the data per the selected template.
  3. Click Save.

Import history

The Import history page contains a table that lists all the processed batches (import tasks) in a project.

Each row item is a batch. By default, the Created on column defines the ordering of the items in the table, displaying the latest batches first. You can sort by any other column by clicking on its name. Clicking on it again will reverse the order, as indicated by the direction of the small arrow to the right of the a column title. You can filter the batches by data set, file name, date of import, or by your batches, or others’ batches.

The table provides the process status for all import batches and displays those that are waiting in queue. You need to refresh the page manually to see most status changes. When a batch status is Error or Done with warnings, you can view the details by clicking the number in the Error / Warnings column.

Deleting data is only possible by deleting the complete batch. Therefore, modifying certain values will require that you delete the whole batch, make modifications to the CSV file, and then re-import the updated file.

It is possible to download the imported CSV file from any batch by clicking on its link in the File column.

Monitoring folders

can monitor folders on the network file system for CSV files matching an assigned combination of project, theme, import template, and data set. This is configured on a per-project basis in the Import menu section, and is accessible only to the system administrator. This functionality is meant for automatically importing CSV files that are generally exported from other devices or systems. You can configure devices to generate CSV files, on schedule, and copy them to the designated folder.

This feature is particularly useful for themes that constantly generate new data and need to be rapidly available in the system for monitoring, such as input for scheduled reports or geohazard assessment calculations. A typical example would be microseismic event data.

Once the you have configured the system, no user interaction is necessary for the data to be verified for QA/QC, imported in the database, and integrated to the data set. If an error occurs during the process, the system administrator, or another designated user, will be notified by email.

To add or update a monitoring file:

  1. In the menu, under the Import menu, click Monitoring folders.
  2. Click Add new.
  3. In the Update monitoring file dialogue box, select the data set, template, and group name to use, then add any relevant tags you would like to link to all the auto-imported data. You can also provide an email address to receive warning and error messages in case of problems during import.
  4. Click Update.

Importing from other databases

It is possible to create customized connections to directly pull data from other site data sources. This is an implementation customized on a per-project basis, and similar to the functionality of monitoring folders, has the advantage of not requiring user interaction, thus limiting potential human errors.
