Параметры проекта

Раздел «Параметры проекта» состоит из последовательности элементов, которые определяют атрибуты и стандартные блоки, составляющие проект Geoscience INTEGRATOR.

Системные администраторы и менеджеры проекта отвечают за определение параметров проекта и по умолчанию только они имеют полномочия на внесение в них изменений. Прочие пользователи имеют возможность просмотра этих параметров для ознакомления.

Параметры проекта находятся внизу списка элементов меню Geoscience INTEGRATOR.

Единицы измерения

Это список единиц измерения, которые используются в проекте. Список задается менеджером проекта.

При создании шаблона импорта, пользователям предлагается указать единицу измерения для каждого свойства. Для этого необходимо выбрать одну единицу из списка, созданного на основе этой таблицы. Важно, чтобы все необходимые единицы измерения были добавлены в таблицу до выполнения импорта данных.

Чрезвычайно важно запланировать какие единицы измерения будут использованы в проекте, чтобы обеспечить постоянство и не допустить, чтобы значения одного свойства были выражены в разных единицах измерения. Например, будьте внимательны, недопустимо задавать значения напряженности массива в кПа, и в МПа. Такая ситуация ограничит функциональность интеграции данных, поскольку инструменты анализа будут воспринимать «напряженность массива в кПа» и «напряженность массива в МПа» как различные свойства.

Приведенные в этом списке единицы измерения не ограничивают единицы, которые вы может использовать в любых файлах, картах, или прочих документах, импортируемых при помощи системы управления данными; они используются только для данных, которые импортируются в базу данных Geoscience INTEGRATOR.

Создавать единицы измерения при помощи кнопки в правой верхней части таблицы «Активные единицы измерения», а также изменять и удалять их могут только пользователи с соответствующими правами доступа.

Примечание: Удаление единицы измерения невозможно, если в базе данных записаны данные с использованием этой единицы измерения.

Классификации

Этот элемент предназначен для импорта данных категоризации в любой теме.

Например, классификация кода породы и входящие в нее категории или классы могут выглядеть так:

  • Код породы:
    • Доломит
    • Известняк
    • Роговик

Задание набора классификаций в проекте обеспечивает проверку согласованности данных на точное соответствие записей существующим классам. Как показано в примере выше, если пользователь попытается выполнить импорт файла данных, содержащего опечатки или неправильное написание слов, процесс импорта будет остановлен и будет выдано сообщение об ошибке с указанием исправить записи, которые не соответствуют заданным классам.

Классификации в проекте обеспечивают соответствие данных, введенных в базу, однотипным классам. Благодаря этому сохраняется единство данных в рамках проекта, что необходимо для проведения анализа.

Эта функция также избавляет от необходимости каждый раз вводить имя класса в шаблон.

Для того чтобы задать классификацию:

  1. В меню Geoscience INTEGRATOR, нажмите пункт Параметры проекта, затем нажмите Классификации.
  2. В таблице Системные классификации задайте имя и описание вашей классификации. Здесь также можно ввести описание.
  3. Нажмите Добавить классификацию.
  4. Задайте классы, как описано ниже:
    1. Нажмите кнопку правки классификаций (кнопка вставки)
    2. В поле «Имя класса» в окне Классы для классификации введите имя класса.
    3. Нажать Добавить класс. (Также можно задать цвет для отображения каждого класса. Для этого нажмите «добавить цвет» и установите цвет по выбору из цветовой таблицы или при помощи ввода RGB-значения цвета).

Если вы планируете добавлять большое количество классов, чтобы избежать ввода свойств классов вручную, можно воспользоваться опцией импорта списка классов из CSV-файла. Для этого нажмите Импорт из файла в окне Классы для классификации и следуйте указаниям. Этот CSV-файл не должен содержать заголовков, а только список имён классов.

Примечание: В программе предусмотрена возможность создания дополнительных классов непосредственно из шаблона импорта (см. раздел «Импорт данных» настоящего руководства). Эти дополнительные классы будут действовать только для данных, которые были импортированы при помощи специального шаблона импорта и не будут добавлены к списку общих классификаций проекта.

Участки

Участки представляют собой задаваемые пользователем пространственные объемы в рамках проекта. Каждый из участков состоит из одного или более трёхмерных многоугольников, имеет смысловое имя и представляет собой область, которую пользователи описывают по имени, а не по координатам. Так, например, в руднике Участок может представлять собой горизонт, выемочную камеру, штрек или околоствольный двор.

Функция «Участки» существенно повышает эффективность пространственных запросов, поскольку в этом случае пользователям не нужно знать/вводить координаты XYZ интересующих областей; вместо координат вводятся названия участков. Такое решение особенно удобно в тех случаях, когда участок состоит из множества многоугольников.

Система постоянно автоматически контролирует состав данных в каждом из участков.

Чтобы создать Соседнюю область:

  1. В меню Geoscience INTEGRATOR, нажмите пункт Параметры проекта, затем нажмите Соседние области.
  2. В таблице «Соседние области» нажмите кнопку Создать соседнюю область.
  3. В диалоговом окне Создать соседнюю область введите имя области. Здесь также можно ввести описание (необязательное).
  4. Нажмите Добавить, затем нажмите Закрыть.

К Участку должно быть добавлено хотя бы одно объемное тело, чтобы идентифицировать уже имеющиеся данные. В таблице Активные участки нажмите число в столбце Кол-во объёмов. Участок может содержать множество объёмов. При нажатии Добавить новый объём предоставляется опция задания ограничивающего объёма или многоугольника с параметром толщины.

Также для создания нового участка существует возможность импорта многоугольника из CSV-файла. Для этого необходимо нажать кнопку Импорт участка в таблице активных участков. Данный CSV-файл должен содержать список координат X,Y с заголовком. Минимальное и максимальное значение координаты Z пользователь вводит в соответствующем поле.

Примечание: При создании или изменении нового объёма система сканирует всю базу данных, чтобы определить какие из трёхмерных данных находятся в этом объёме. Такая операция может занимать определенное время. Аналогично, в процессе импорта выполняется сканирование всех новых данных с целью отслеживания, на каких участках располагается каждая точка.

Участки состоят из одного или более трёхмерных объёмных тел, где границы каждого объемного тела обозначены ограничивающим объёмом с координатами XYZ или многоугольником с заданным диапазоном значений по оси Z.

  • Ограничивающий объём: Параллелепипед, заданный координатами юго-запад и северо-восток.
  • Многоугольник: Состоит из бесконечного числа точек XY, которые объединены для создания закрытого двухмерного многоугольника, в котором последняя точка соединена с первой, завершая построение многоугольника. Чтобы трансформировать двумерный многоугольник в трёхмерное объёмное тело, необходимо задать максимальные и минимальные значения для оси Z.

Примечание: Участки независимы друг от друга и могут перекрываться. Поэтому любая из точек может принадлежать нескольким участкам.

Горизонты рудника

Этот элемент предназначен для определения максимальной и минимальной отметки высоты для каждого горизонта рудника в проекте. В этом параметре можно добавлять и удалять горизонты при наличии соответствующих полномочий пользователя. Горизонты рудника используются только в разделе Сборка модели рудника программы Geoscience INTEGRATOR.

Рабочие смены

Рабочие смены представляют собой заданные отрезки времени в течение 24-часового периода, что позволяет задавать активные временные интервалы в рамках проекта. Параметр «Рабочие смены» облегчает процесс построения запросов по критерию времени, поскольку в этом случае не требуется вводить специальных значений начала и окончания временного интервала. Вместо этого пользователь может ввести только заданное имя смены.

Типичным примером может быть создание утренней, дневной и ночной смены, в зависимости от графика подземных работ.

Мы рекомендует при создании смен не допускать накладок смен друг на друга, хотя система Geoscience INTEGRATOR не ограничивает и не контролирует этот параметр. Пользователи могут изменять имена смен, описания, а также значения времени начала и окончания смены. Для создания, изменения и удаления рабочих смен необходимы соответствующие права пользователя.

Чтобы задать смены в проекте:

  1. В меню Geoscience INTEGRATOR, нажмите пункт Параметры проекта, затем нажмите Рабочие смены.
  2. В таблице Активные смены нажмите Добавить новую смену.
  3. В диалоговом окне Добавить новую смену:
    1. Введите имя и описание смены.
    2. Задайте время начала и окончания смены.
  4. Нажмите Добавить, затем нажмите Закрыть.

 

Типы файлов

Этот элемент представляет собой список файлов, которые могут использоваться в качестве параметра на страницах «Диспетчер файлов» и «Карты/планы/разрезы». В этом окне вы можете добавлять, изменять и удалять типы файлов, при наличии соответствующего доступа.

Назначенные пользователи

Руководители проекта или системные администраторы имеют возможность контролировать роли пользователей в рамках отдельного проекта. На этой странице содержится список учетных записей пользователей в рамках активного проекта, включая имя пользователя, адрес электронной почты, организацию и назначенную роль. Системные администраторы и руководители проекта могут удалять пользователей при нажатии на красный X в конце строки, или добавлять пользователей в проект путём выбора имён из раскрывающегося списка внизу таблицы и выбора ролей, и подтверждения нажатием кнопки Сохранить.

В соответствии с отведённой ролью выдаются определённые полномочия для работы в системе. Роли присваиваются в зависимости от проекта, поэтому пользователи могут иметь разные роли (уровни доступа) в различных проектах.

Примечание: Чтобы изменить роль пользователя необходимо удалить текущую роль и присвоить новую.
Примечание: Для того чтобы назначить роли пользователей в проекте, пользователи должны быть предварительно внесены в систему. Добавление пользователей в систему может выполнять только администратор системы.
