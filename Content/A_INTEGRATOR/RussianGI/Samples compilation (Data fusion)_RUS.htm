Объединение данных

Объединение данных представляет собой интеграцию данных из разных источников с целью получения информации, которая имеет большую пользу, чем могут обеспечить отдельно взятые наборы данных. В системе Geoscience INTEGRATOR объединение данных доступно для 3D данных, для данных проб и для 4D данных модели рудника.

В этом процессе Geoscience INTEGRATOR предлагает мощный набор инструментов по поддержке анализа комплексных и сложнопросматриваемых отношений внутри данных. Инструмент объединения данных компилирует данные из различных источников, и таким образом экономит время и сокращает риск ошибки в случае выполнения работ по объединению данных вручную.

Объединенные данные могут стать входными данными для алгоритмов машинного обучения, что требует приведения данных к одной опорной точке, при распределении значений в пространстве и времени.

В среде Geoscience INTEGRATOR процесс объединения данных состоит из двух элементов:

  • Сборка проб: Сборка данных пробы из различных наборов данных в различных темах, которые ссылаются на один и тот же идентификатор пробы.
  • Сборка модели рудника: Сборка многочисленных свойств, относящихся к геотехническим рискам в точках модели рудника в виде единственной ссылки типа дата-время.

 

Сбор проб

Пробы различаются по сущности и имеют различные формы. Зачастую анализ пробы выполняется при помощи различных инструментов и методов и преследует различные цели. Geoscience INTEGRATOR позволяет импортировать и ассоциировать данные, относящиеся к пробам во множествах наборов данных и темах, в зависимости, как от сущности свойств пробы, так и от того, каким образом эти свойства относятся к теме. Сборка проб является мощным инструментом для просмотра всех данных, ассоциированные с пробой, по множеству наборов данных и тем.

Для данных, относящиеся к пробам предусмотрены три темы и одна подтема:

  • Пробы
  • Геохимия и минералогия
  • Характеристики пород
  • Скважинный интервал (подтема)

Например, проба бурового керна может содержать:

  • Геологическое описание, импортированное в скважинных интервалах с соответствующими данными расположения (положение в скважине от и до с известными данными местоположения устья скважины и данных траектории).
  • Данные плотности, пористости, удельного электрического сопротивления и магнитной восприимчивости импортируются в теме «Физические свойства».
  • Результаты количественного анализа всех элементов и минералов содержатся в теме «Геохимия и минералогия».

При работе с данными проб объединение данных возможно, поскольку для каждой темы, связанной с пробами, необходим идентификатор пробы. Компонент сборки проб позволяет формировать запрос ко всем данным, которые ссылаются на идентификатор пробы, независимо от темы или набора данных.

Пользовательский интерфейс модуля сборки проб состоит из двух разделов:

  • Вкладка Обзор
  • Вкладка Диаграмма | Таблица

На вкладке Обзор по идентификатору пробы составляется сводка всех свойств (рассортированных по наборам данных и темам) и всех меток и файлов, сопоставленных с данным идентификатором пробы, а также материал пробы и доступные данные местонахождения.

Во вкладке Обзор могут быть выполнены следующие действия:

  • Применение фильтра к группе путем ввода условия поиска в поле Имя пробы (запрос выполняется только по идентификаторам пробы и не затрагивает свойства, метки и т.п.). Также можно установить флажок в поле «Идентификатор пробы» и нажать Фильтр.
  • Просмотр всех свойств (упорядоченных по наборам данных и темам) для выбранного идентификатора пробы. Можно нажать на набор данных и перейти в окно Диспетчера базы данных, где будет отображаться этот конкретный набор данных.
  • Просмотр Меток, сопоставленных с идентификаторами проб. При нажатии на имя метки будет выполнен переход непосредственно на страницу меток, где эта метка будет выделена, для просмотра всех наборов данных, ассоциированные с этой меткой.
  • Просмотр Файлов, сопоставленных с идентификатором пробы независимо от темы. При нажатии на имени файла, файл будет загружен.
  • Просмотр, а также ввод или изменение информации о местоположении пробы выполняется при нажатии кнопки Идентификатор пробы. Если данные о местоположении еще не введены, вы можете ввести координаты XYZ или выбрать Скважину и ввести значения Интервал от и Интервал до.
Примечание: Если в последнем столбце в строке пробы стоит красный X, это означает, что в базе данных нет записей, сопоставленных с этой пробой. Ссылки на метки и файлы сохраняются. В таком случае идентификатор пробы выступает в роли метки-заполнителя. При нажатии X выполняется удаление записи без возможности восстановления.

Вкладка Диаграмма | Таблица позволяет просматривать данные из различных наборов данных и тем в общей диаграмме и таблице данных.

На этой вкладке предлагаются такие же опции применения фильтров и настройки таблиц и диаграмм, как и в окне Диспетчер базы данных. В то же время здесь доступны свойства из всех наборов данных и тем, которые относятся к пробам, что позволяет анализировать их вместе и исследовать перекрестные наборы данных или отношения, проходящие через множество тем. Например, вы можете просмотреть диаграмму разброса содержания железа (Fe) в пределах темы «Геохимия и минералогия» в противопоставлении плотности пород из темы «Физические свойства» и провести наблюдение прямой корреляции между этими двумя свойствами. Также возможен просмотр данных, собранных по идентификатору пробы, в таблице.

 

Сборка модели рудника

Модуль Сборка модели рудника приводит все данные модели рудника к общей опорной точке при распределении значений в пространстве и времени. Эта возможность объединения данных является принципиальным качеством для выполнения оценки геотехнических рисков.

Этот модуль состоит из трех вкладок:

  • Обзор
  • Планы горизонтов рудника
  • Составленная таблица

Примечание: Тема Подземный (с моделями рудника) объединяет и сводит вместе распределенные данные. По умолчанию применяется расстояние распределения 10 сантиметров, что означает, что точки, расстояние между которым менее 10 сантиметров, будут объединены в одну точку.

Вкладка «Обзор»

На этой вкладке для каждого набора подземных данных модели рудника в проекте отображается поле, в котором представлена суммарная информация по количеству точек, импортированным свойствам и свойствам, которые рассчитываются системой.

При нажатии на поле указателем мыши поле разворачивается и открывает детальный список свойств, который может представлять следующую информацию:

  • Импортированные свойства: Импортируемые пользователем свойства, которые не обновляются системой. Как правило, это фиксированные свойства, которые не будут изменяться со временем, например, дата отработки.
  • Свойства, которые рассчитываются системой: Результат различных расчетов, выполненных с использованием других данных и/или файлов в проекте. Выполнение расчетов может быть запланировано или активировано вручную. Для получения дополнительной информации см. главу Аналитика геотехнических рисков.



 

Примечание: Различные индексы геотехнических рисков не включены в этот раздел в качестве свойств Модели рудника.

 

Вкладка «Планы горизонтов рудника»

На этой вкладке представлена схема расположения точек модели рудника для каждого рудничного горизонта.

Примечание: Перечень горизонтов рудника и их минимальные и максимальные отметки высот содержатся в настройках проекта. Этот перечень могут редактировать только пользователи с особыми полномочиями.

Чтобы отобразить план горизонта рудника:

  1. Разверните панель Фильтры в правой стороне экрана и выберите один набор данных.
  2. Выделите те Горизонты рудника, которые необходимо просмотреть (один или несколько).
  3. Нажмите указателем мыши кнопку Применить изменения, чтобы просмотреть данные.

Нажмите раздел Границы, чтобы настроить границы плана горизонта. В результате этой операции новые границы будут применены ко всем выделенным горизонтам рудника.

Вы можете выбрать свойство из раскрывающегося меню Радиус , чтобы настроить размер точек в соответствии с значениями свойств. Если значение вводится в поле номера Пороговое значение, то размер точки будет установлен в зависимости от того, будет ли значение точки выше или ниже порогового.

Кнопки Значение радиуса позволяют настроить корреляцию значений с величиной радиуса.

Вы можете выбрать свойство из раскрывающегося меню Цвет , чтобы настроить цвет точек в соответствии с значениями свойств. Предусмотрена также опция применения к цветовой кодировке логарифмической шкалы.

Поле На сегодняшний день предназначено для определения конкретного момента «дата-время» для создания запроса и по умолчанию имеет значение момента обращения к функции.

Кнопки Цветовой спектр предназначены для настройки изменения цветовой шкалы от синего к красному или наоборот.


Примечание: Если в секторах Величина радиуса и Цветовой спектр не будет задано значение На сегодняшний день для свойства момента времени или временного интервала, будут применены наиболее актуальные значения. Если значение будет задано, то будут применены наиболее актуальные значения до заданной даты.

 

 

Вкладка «Составленная таблица»

Эта вкладка представляет собой моментальный снимок всех точек в наборе данных в определенный момент времени. На ней отображаются все свойства, выбранные вами из списка для всех точек.

 

Чтобы отобразить составленную таблицу:

  1. Разверните панель Фильтры в правой стороне экрана и выберите один набор данных.
  2. Выделите столбцы, которые необходимо отобразить.
  3. Задайте дату и время, по которым вы хотите составить данные. (Дополнительно, запрос можно ограничить отдельными Участками или Горизонтами рудника.)
  4. Нажмите указателем мыши кнопку Применить изменения, чтобы просмотреть данные.

Для каждого свойства моментальных данных, в окне Составленная таблица будет отображено значение, которое точно соответствует указанному дате-времени или ближайшему дате-времени до указанного.

Для каждого свойства интервала времени в окне Составленная таблица будет отображено значение, если указанная дата-время приходится на интервал времени, для которого в системе есть данные.

Поскольку свойства статических данных не изменяются со временем, в окне Составленная таблица всегда будет выдавать одно и то же значение для каждой точки модели рудника независимо от указанной даты и времени.

Поле На сегодняшний день предназначено для определения конкретного момента для создания запроса и по умолчанию имеет значение момента обращения к функции.

При указании параметров Участки и/или Горизонты рудника для выборки результатов запроса, система собирает данные по ближайшим датам, на которые по запрошенным точкам имеются данные. Результаты могут отличаться от наиболее актуальных данных, которые относятся к всем точкам Модели рудника.
