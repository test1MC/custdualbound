Импорт данных

В этой главе описывается методика импорта данных для сохранения в виде записей в базе данных. Информация об импорте файлов, карт и документов представлена в главе 6.

Стандартный способ импорта данных в систему Geoscience INTEGRATOR состоит в предоставлении данных в виде CSV-файла и последующей обработке файла в настраиваемых шаблонах импорта для правильного упорядочивания данных в базе данных. Это может быть выполнено при помощи пользовательского интерфейса или при помощи системы контрольных папок.

Помимо этого, в отдельных темах предусмотрена опция прямого импорта из файлов особых форматов (файлы GOCAD) или ручного ввода данных. Также может быть выполнено получение данных из баз данных других объектов при помощи специально настраиваемых логических блоков объединения.

 

Импорт при помощи CSV-файла

Данный инструмент импорта использует файлы в формате CSV, где в первой строке содержатся имена свойств.

Для того чтобы программа знала, какую информацию следует импортировать из файлов CSV и куда и как поместить ее, в базе данных Geoscience INTEGRATOR, необходимо создать шаблоны импорта. Для каждой темы предусмотрены собственные соответствующие варианты шаблонов.

Применение системы шаблонов импорта - это удобный способ, который позволяет импортировать любой тип данных, поскольку для этого не требуется, чтобы данные соответствовали какому-либо определенному или заданному типу или формату.

Подготовка шаблонов импорта:

  1. В списке Активная тема в верхней части пользовательского интерфейса, выделите тему, которая наиболее соответствует данным, которые необходимо импортировать.

  2. В меню Geoscience INTEGRATOR, нажмите пункт Импорт/ Импорт данных / Файл CSV.

Примечание: В зависимости от темы, опции шаблонов могут немного отличаться в силу специфики темы (или структуры данных). Необходимо выбрать и создать шаблоны на основе характеристик ваших данных. Поэтому, в зависимости от темы, предусмотрено от одного до пяти вариантов структуры шаблона.
  1. Выберите шаблон, который соответствует вашему типу данных. На этом этапе можно создать новый шаблон или выбрать из уже имеющихся. Чтобы создать новый шаблон, нажмите Создать новый.
  2. Чтобы создать новый шаблон, задайте имя вашего шаблона и нажмите Создать.

Обязательные и пользовательские свойства

В созданном новом шаблоне необходимо задать свойства данных (столбцы CSV-файла) для импорта как обязательное или пользовательское.

Обязательные свойства являются минимальным набором свойств для создания применимой записи для активной темы. Например, тема импорта точек (напр. пунктов наблюдения) предусматривает наличие всех трех координат для размещения точки в пространстве. А тема для документированных событий (напр. инцидентов) предусматривает наличие трёхмерных координат и параметра дата-время. По этой причине система не позволит импортировать записи (строки CSV-файла) в тему с отсутствующими обязательными свойствами.

Имена обязательных свойств жестко зафиксированы в системе. Если эти обязательные столбцы называются иначе в импортируемом CSV-файле, вы можете их переименовать в файле или указать в шаблоне имя столбца CSV-файла, в котором находятся соответствующие данные. Эта операция называется «присвоение» свойств.

Присвоение столбцов CSV-файла:

  1. В столбце Имя нажмите свойство, которое необходимо присвоить другому имени в CSV-файле.
  2. В диалоговом окне Свойство: имя, замените текст в поле Присвоено таким образом, чтобы он соответствовал имени столбца СSV-файла.
  3. Нажмите Обновить.

Примечание: Типы данных и единицы измерения для обязательных свойств неизменяемы.

Пользовательские свойства предусмотрены для других, необязательных импортируемых данных. Так, например, в теме «Буровзрывные работы», помимо обязательных свойств x, y, z и дата-время можно указать причину (подготовительные работы, очистные работы, и т.д.), наименование очистной выработки, тип и количество взрывчатого вещества, тип детонатора, или фамилию взрывника.



 

Для того, чтобы задать пользовательское свойство:

  1. В таблице Свойства нажмите Добавить новое свойство.
  2. В диалоговом окне Добавить новое свойство введите имя свойства из вашего CSV-файла, которое требуется добавить в шаблон.
Примечание: Имя свойства не является чувствительным к регистру, в шаблоне оно должно быть однозначно определяемым. Его можно использовать в каком-либо другом шаблоне в этой же теме, при условии, что тип данных и единица измерения являются одинаковыми (имя свойства может иметь только один тип и одну единицу измерения в рамках одной темы).
В отношении использования одинакового имени для свойств в разных темах ограничений нет, независимо от типа и единицы измерения.
Система записывает в базе данных введенные в этом окне имена и использует их для идентификации данных повсеместно в пользовательском интерфейсе и отчетах. Рекомендуется очень внимательно подбирать имена свойств.
  1. Задайте тип свойства из следующих шести доступных типов:
Property type Definition
Буквенно-цифровой знак Используется для текстовых строк произвольной формы, состоящих из текста и/или цифр. Обратите внимание, что если в этих строках присутствуют запятые, необходимо поставить двойные кавычки (“ и “) в начале и в конце строки, чтобы средство импорта не воспринимало эти запятые как запятые, которые разделяют значения в CSV-файлах. .
Двоичное число Используется для двоичных данных (например, вкл./выкл., руда/порода, истина/ложь). Обратите внимание, что при выборе этой опции вам будет предложено ввести оба значения.
Классификации
  • Классификации устанавливаются заранее на уровне проекта или определяются в ходе подготовки шаблона импорта.
  • При выборе классификации уровня проекта на экране будет отображен список классов.
  • К этому списку могут быть добавлены новые классы, но эти добавления будут относиться только к тем данным, которые готовятся к импорту.
  • Классификация уровня проекта не будет обновлена с учетом добавленных классов.
  • Если не была выбрана опция классификации уровня проекта, просто введите имена классов с разделением запятыми.
  • Здесь необходимо указать являются ли классы чувствительными к регистру или нет; если классы чувствительны к регистру, то значения CSV-файла должны точно соответствовать буквам в верхнем и нижнем регистре из списка классов.
Дата-время

Необходимо указать формат даты, используемый в файле CSV.

  • Дата может включать в себя время в формате чч:мм:сс или только в формате чч:мм. Данный формат не настраивается.
  • Обратите внимание, если дата не включает в себя значений времени, то система по умолчанию устанавливает значение 00:00:00.

Укажите, разрешать или не разрешать импорт данных с датой, которая находится в будущем (значение даты превышает текущую системную дату).
Число с плавающей точкой Для числовых значений с десятичными знаками.
Целое число Для числовых значений без десятичных знаков.

Если ответ да, то исходные данные, подлежащие импорту при помощи этого шаблона должны содержать значения для этого свойства для всех импортируемых записей, в противном случае импорт даст сбой (Ошибка). (Этот параметр полезен при определении недостающих данных. Если обязательное поле установлено на значение нет и для этого свойства в CSV-файле присутствуют незаполненные записи, система не остановит процесс импорта. Пустые или незаполненные записи в данных отмечены в базе данных как значения без данных (NDV).

В таком случае необходимо ввести соответствующее значение заполнителя, чтобы система распознала его как значение без данных (например, -99999 согласно стандарту GOCAD).

  4. Нажмите Сохранить и повторите процесс добавления необходимых свойств.

 

Для определенных тем или типов данных система обеспечивает следующие настройки:

  • Значение свойства ниже порога чувствительности: Применяется только в теме «Геохимия и минералогия». В этом свойстве вводится Значение ниже предела чувствительности (UDL). Эти значения указываются в лабораторных сертификатах. В поле Формат введите точно текстовую строку, которая используется в CSV-файле, и в поле Значение введите это значение. Пример: Формат: <0,01, Значение: 0,01.
  • Системные метки: См. раздел «Системные метки» в главе«Метки». Используется только в некоторых шаблонах: Пробы, Скважины, Интервальные данные, Скважины, Каротажные данные точек бурения скважин, и Точки наблюдения.
  • Правила контроля качества: В целях контроля качества, а также для свойств типа «число с плавающей точкой/целое число/дата-время» можно задать минимальное и максимальное значение для проверки данных. Если какое-либо из значений не пройдёт проверку, процесс импорта выдаст сбой (Ошибку).
Примечание: Количество пользовательских свойств, которые могут быть добавлены, не ограничено.

В качестве альтернативного варианта можно нажать кнопку Считать заголовки, которая позволяет выбрать на жестком диске CSV-файл, как правило, тот, который вы планируете импортировать при помощи этого шаблона. Программа Geoscience INTEGRATOR просканирует файл и создаст пользовательские свойства на основе заголовков выбранного CSV-файла. Тем не менее, необходимо будет открыть настройки каждого свойства, чтобы установить соответствующий тип данных, единицу измерения, и прочие параметры. Важно не забыть нажать кнопку «Сохранить», прежде, чем переходить к следующему свойству.

Система предоставляет возможность создавать копию шаблона. Чтобы создать копию шаблона:

  1. В диалоговом окне Создать копию шаблона импорта задайте новое имя копии шаблона.
  2. Задайте Необязательные поля, которые будут отображаться в зависимости от активной темы.
  3. Нажмите Создать копию шаблона импорта. Автоматически выполнится переход новый скопированный шаблон, где можно редактировать необходимые вам параметры

Специфика типа тем

В зависимости от темы и шаблона, при импорте данных пользователю доступны несколько действий. В следующих таблицах перечислены доступные действия для каждого шаблона. Обратите особое внимание на Требования к однозначности, где указано сочетание свойств, которые должны быть однозначными для правильного выполнения импорта, поскольку система не может принять более одного значения для свойства (для каждой записи).

Темы для данных типа точек

Как показано в таблице выше, при готовности импортировать новые данные с помощью шаблона темы для данных типа точек, необходимо присвоить шаблон набору данных. Это может быть либо имеющийся набор данных, либо вы можете создать здесь новый (если у вас есть соответствующий уровень разрешения).

Кроме темы «Микросейсмические датчики», все темы требуют при импорте также присвоения новым данным имени группы. Это может быть либо имеющийся набор данных, либо вы можете создать здесь новый, при условии если у вас есть соответствующий уровень разрешения.

Правила однозначности выполняют проверку данных при импорте. Нельзя иметь одну и ту же комбинацию значений для этих свойств более одного раза в одном наборе данных или даже в другом наборе данных в пределах одной и той же темы и проекта.

Во всех темах типа точек (за исключением темы «Микросейсмические датчики») присутствует свойство Группа, которое предназначено для разделения импортируемых данных для выбора по фильтрам в будущем. Так например: Импорт данных, собранные в точках, которые могут периодически пересчитываться, может быть выполнен с новым именем группы, которое отражает каждый пересчет.

Эта функция также позволяет осуществлять импорт одной и той же комбинации координат (одна и та же точка), если они присвоены другому имени группы. Так например: При использовании Точек наблюдения для структурных измерений, сбор результатов которых выполняется в одном и том же месте, например, осевая линия складки и осевая плоскость, им необходимо будет задать однозначные имена групп.

Для темы «Микросейсмические датчики» комбинация идентификатора датчика и координат x, y, z должна быть однозначной в пределах одной темы и одного проекта, независимо от набора данных.

Примечание: Имена групп должны быть однозначными в пределах одного набора данных. Следовательно, имя группы может использоваться повторно в другом наборе данных.

Темы типа событий

Как показано в таблице выше, при готовности импортировать новые данные с помощью шаблона темы для данных типа событий, необходимо присвоить его набору данных. Это может быть либо имеющийся набор данных, либо вы можете создать здесь новый, при условии если у вас есть соответствующий уровень разрешения.

Правила однозначности выполняют проверку данных при импорте. Нельзя использовать комбинацию данных дата-время, x, y, z более одного раза в одной теме и в одном проекте, независимо от наборов данных.

Темы, относящиеся к пробам

Как показано в таблице выше, при готовности импортировать новые данные с помощью шаблона темы, относящейся к пробам, необходимо присвоить его набору данных. Это может быть либо имеющийся набор данных, либо вы можете создать здесь новый, при условии если у вас есть соответствующий уровень разрешения.

Правила однозначности выполняют проверку данных при импорте. Обратите внимание, что особенность тем, относящимся к пробам, состоит в том, что Правила однозначности не ограничиваются данной темой, а действуют для всех тем, связанных с пробами. Это означает, что сочетание идентификатора пробы и значения свойства должно быть однозначным во всех темах, относящихся к пробам и во всех наборах данных. Это необходимо для того, чтобы обеспечить, чтобы в модуле сборки пробы не появилось конфликтующих или дублированных данных.

Такая специальная структура данных в темах, относящихся к пробам, предназначена для того, чтобы позволить импортировать данные из различных областей отдельно, но при этом сохранять их связь посредством идентификатора пробы.

Так, например, геолог может импортировать идентификатор пробы с координатами, датой отбора и фамилией отборщика в теме «Пробы». Затем впоследствии могут быть импортированы результаты измерений удельного сопротивления, магнитной восприимчивости и удельного веса в теме «Физические свойства». И наконец, когда будут получены из лаборатории результаты количественного анализа, они могут быть импортированы в тему «Геохимия и минералогия».

Примечание: Свойство скважинного интервала может быть привязано к темам, связанным с пробами при помощи выбора параметра Да внизу поля настройки параметров свойств для запроса Это идентификатор пробы.

Темы, связанные со скважинами

Как описано в вышеприведенной таблице, для импорта данных скважин предусмотрено пять шаблонов.

При помощи шаблона Скважины выполняется импорт данных устья скважины (идентификатор скважины, координаты и длина). Для импортируемых скважин необходимо задать набор данных. Это может быть либо имеющийся набор данных, либо вы можете создать новый из этого шаблона, если у вас есть соответствующий уровень разрешения. При необходимости, вы можете ассоциировать метки и файлы со скважинами. Согласно Правилу однозначности однозначным должен быть только идентификатор скважины независимо от наборов данных.

После импорта, как минимум, одной записи идентификатора скважины становятся доступными опции:

  • Данные траектории скважин для документирования замеров отклонения, выполненных в скважине на заданных интервалах. (Эти данные будут использованы для просмотра скважины в приложении трёхмерной визуализации).
  • Данные интервалов скважин для свойств, относящихся к интервалу скважины, который вы задали по параметрам расстояния «от» и «до».
  • Каротажные данные точек бурения для свойств, которые были задокументированы для конкретной глубины ствола скважины (не считая измерений отклонения траектории).
  • Статические данные скважин используются для любых дополнительных свойств, которые относятся ко всей скважине, например, начало и окончание бурения, название подрядчика по бурению, имя лица, записавшего эти данные и т.д.

К данным, которые импортированы при помощи этих четырех шаблонов, не могут быть привязаны наборы данных, метки или файлы. Они ссылаются на идентификатор скважины, который является объектом, уже упорядоченным в наборы данных, и к нему уже могут быть привязаны метки и файлы. См. дополнительную информацию о сочетаниях свойств, которые должны быть однозначными для успешного импорта данных в Правилах однозначности в таблице выше.

Объекты с данными, связанными со времене

 

Как описано в вышеприведенной таблице, для импорта данных различной структуры во времязависимые темы предусмотрено четыре шаблона. Во всех из этих тем предусмотрен первый шаблон для импорта базовых обязательных данных для каждого типа объекта: пункты наблюдений для тем «Деформация грунта» и «Работа стационарных установок», Тензочувствительные элементы для темы «Напряженность массива», единицы для темы «Подвижное оборудование». Для импортируемых объектов необходимо задать набор данных. Это может быть либо имеющийся набор данных, либо вы можете создать новый из этого шаблона, если у вас есть соответствующий уровень полномочий. При необходимости, вы можете привязывать метки и файлы к любой записи. Согласно Правилу однозначности однозначным должен быть только идентификатор объекта (в пределах одного проекта и в теме, даже если находится в разных в наборах данных). Например, свойство идентификатора пункта наблюдений со значением STA01 не может находиться в двух разных наборах данных в теме «Деформация грунта».

После выполнения, как минимум, одного из вышеприведенных пунктов, становятся доступными опции:

  • Шаблон моментальных данных темы для свойств, которые происходят в определенный момент дата-время (например, показания приборов). Примером моментальных данных может быть стандартный временной ряд: данные записываются как набор отдельных моментов времени. Моментальные данные в системе Geoscience INTEGRATOR представляют собой обобщение стандартного временного ряда, в котором время измерений не обязательно должно быть отмечено регулярными интервалами, а свойства могут быть выражены нечисленными значениями.
  • Шаблон данных темы определенного интервала для свойств, относящихся к интервалу времени, ограниченному параметром дата-время «от» и «до» (например, состояние, среднее значение показаний, и т.д.). Интервальные данные в системе Geoscience INTEGRATOR представляют собой расширение моментальных данных, в которых каждое измерение имеет связанный временной интервал (время начала и время окончания), в отличие от измерений, связанных с единичным моментом времени.
  • Шаблон статических данных темы для любых дополнительных свойств, не зависящих от темы, а относящихся к объекту в общем (например, производитель, тип, дата установки и т.д.).

К данным, которые импортированы при помощи этих трех шаблонов, не могут быть привязаны наборы данных, метки или файлы. Они ссылаются на идентификатор объекта, который является объектом, уже упорядоченным в наборы данных, и к нему уже могут быть привязаны метки и файлы. См. дополнительную информацию о сочетаниях свойств, которые должны быть однозначными для успешного импорта данных в Правилах однозначности.

Обратите внимание, что в Справочной таблице по созданию шаблонов для различных тем, в отличие от других тем аналогичного типа (объекты с данными, связанными со временем), шаблон единицы подвижного оборудования не включает свойств x, y, z. Причина этого в том, что положение подвижных единиц оборудования может изменяться во времени, как и прочие свойства моментальных данных. Поэтому в этой теме предусмотрено два шаблона моментальных данных: один включает в себя данные местоположения, а другой не включает. Последний может быть использован, например, при необходимости регистрации уровней жидкостей, а не положения единиц оборудования.

Связанные с моделями рудников

Как описано в вышеприведенной таблице, для импорта данных различной структуры в теме «Подземные данные» предусмотрено четыре шаблона. Несмотря на то, что структура подобна другим структурам для объектов с данными, связанными со временем, её правила проверки отличаются.

Для темы «Подземные данные» объектом являются точки. Для импорта координат подземных точек используется шаблон «Подземные точки». Для импортируемых точек необходимо задать набор данных. Это может быть либо имеющийся набор данных, либо вы можете создать новый из этого шаблона, если у вас есть соответствующий уровень полномочий. При необходимости, вы можете привязывать метки и файлы к набору данных.

В соответствии с Правилами однозначности, сочетание набора данных и координат x, y, z должно быть однозначным. Это означает, что вы можете импортировать точку (единственное сочетание x, y, z) в различные наборы данных в одной и той же теме (в отличие от прочих объектов с темами, связанными со временем, которые разрешают присутствие только одного объекта в одном наборе данных).

Кроме того, в этой теме применяется ограничение по близкому размещению. В наборе данных не могут содержаться точки, которые находятся ближе 10 см от соседней точки. Свойства таких близколежащих точек сводятся в одну точку. Если при импорте сведенных точек выполняется попытка импортировать уже заполненное значение для свойства, система сохраняет только первое значение, и игнорирует остальные.

После выполнения, как минимум, одного из вышеприведенных пунктов, становятся доступными опции:

  • Шаблон моментальных подземных данных для свойств, задокументированных в определенный момент дата-время (например, Плотность микросейсмических событий).
  • Шаблон интервальных подземных данных для свойств, относящихся к интервалу времени, ограниченному параметром дата-время «от» и «до» (например, Тип крепления).
  • Шаблон статических подземных данных для любых дополнительных свойств, относящихся к объекту в общем (например, Дата отработки).

Одни и те же подземные точки могут встречаться в более чем одном наборе данных. Поэтому, чтобы система ссылалась на правильные точки, при импорте моментальных, интервальных или статических данных в этой теме необходимо задать набор данных. Во время импорта при помощи этих трёх шаблонов привязка меток и файлов не работает, поскольку эта функция активна только на уровне объекта (Подземные точки).

См. дополнительную информацию о сочетании свойств, которые должны быть однозначными для успешного импорта данных при помощи этих шаблонов в Правилах однозначности. Особенностью этой темы является возможность добавлять новые свойства к уже существующим записям, как отмечено наличием свойства в правиле однозначности. Так например, при попытке импорта значения свойства в подземные статические данные (набор данных, x, y, z, свойство), для этого свойства в этой точке не должно быть ранее записанного значения (x, y, z) в этом наборе данных.

Еще одним специфическим свойством темы Модели рудника > Подземные данные является способность создавать отсутствующие точки при импорте моментальных, интервальных или статических данных. Следовательно, в отличие от других тем для объектов с в данными, привязанными ко времени, объекты не должны существовать в системе перед тем, как импортируемые данные начнут на них ссылаться. Также возможно, что возникнет необходимость присвоить набор данных при использовании моментального, интервального или статического шаблона. Создаваемые таким образом точки должны будут удовлетворять требования ограничения по близкому размещению.

Триангулированные поверхности 3D

Как указано в таблице выше, эти темы предназначены для документирования триангулированных поверхностей, которые состоят из точек (вершин) и треугольников. Импорт которых данных этих объектов должен выполняться отдельно при помощи соответствующих шаблонов.

Все шаблоны для трёхмерных триангулированных поверхностей требуют назначения набора данных и группы и разрешения на привязку меток и файлов. В то же время шаблоны треугольников не разрешают создания новых наборов данных и групп, поскольку данные треугольников должны ссылаться на уже существующие точки. Поскольку одинаковые места нахождения точек могут быть дублированы путем присвоения новой группы, необходимо указать эту группу для того, чтобы треугольники ссылались на правильный набор точек

Примечание: Имена групп должны быть однозначными в пределах одного набора данных. Поэтому имя группы может использоваться повторно в другом наборе данных.

 

Импорт файлов GOCAD

В темы, содержащие данные трёхмерных поверхностей (как правило, состоящие из точек и треугольников), данные могут быть импортированы непосредственно из файла 3D GOCAD. В настоящее время поддерживаются только файлы .ts (файлы триангулированных поверхностей GOCAD).

Процедура импорта файла GOCAD:

  1. В окне Импорт данных, нажмите Файл GOCAD.
  2. Выберите Выбрать файл, и в проводнике найдите необходимый файл типа .ts.
  3. Присвойте набор данных, и метки, а также свойства, которые содержатся в файле. Для обязательных свойств можно выбрать единицы измерения и отображаемые имена. Для остальных пользовательских свойств можно задать типы данных, единицы и заполнители значений без данных, так, как при подготовке шаблонов.
  4. Нажмите Загрузить файл.

 

 

 

 

Ручной ввод данных

Ручной ввод данных позволяет импортировать отдельные записи по одной без необходимости применения CSV-файла, что может потребовать больше времени, если необходимо импортировать небольшое количество записей. Обратите внимание, что в настоящее время система поддерживает ручной ввод данных только для темы «Инциденты».

Ввод данных вручную:

  1. В окне Импорт данных, нажмите Ввод данных вручную.
  2. Выберите тему, и затем имеющийся шаблон в этой теме в этом же проекте. Откроются поля, в которые можно ввести значения для обработки данных согласно выбранному шаблону.
  3. Нажмите Сохранить.

 

Импорт журнала

На странице Журнал импорта содержится таблица, в которой представлен список всех обработанных пакетов (задач на выполнение импорта) в проекте, и включены события во всех темах.

Каждый элемент строки представляет собой обработанный пакет. По умолчанию, столбец Дата создания определяет порядок элементов таблицы, при этом последние обработанные пакеты отображаются первыми в списке. Возможна сортировка данных по любому другому столбцу, для этого нужно нажать на заголовок столбца. При повторном нажатии будет изменен порядок отображения согласно направлению стрелки справа от заголовка столбца. Выборка данных может быть выполнена по набору данных, имени файла, дате импорта, а также только по собственным пакетам или по пакетам других пользователей.

Таблица отображает состояние процесса обработки для всех пакетов импорта и отображает пакеты, которые находятся в очереди ожидания. Для просмотра последних изменений состояния необходимо обновить страницу в ручном режиме. Если состояние обработки пакета отображается как Ошибка, или Выполнено с предупреждениями, то с деталями можно ознакомиться, нажав на номер в столбце Ошибка / Предупреждения.

Удаление данных возможно только при удалении пакета целиком. Следовательно, для изменения определённых значений потребуется удалить целый пакет, внести изменения в CSV-файл, а затем повторно импортировать обновлённый CSV-файл.

Загрузка импортируемого CSV-файла может быть выполнена при нажатии на ссылку соответствующего файла в столбце Файл.

Контрольные папки

В системе Geoscience INTEGRATOR предусмотрена возможность отслеживания в файловой системе сети CSV-файлов по заданному сочетанию значений проекта, темы, шаблона импорта и набора данных. Эта функция настраивается для каждого проекта отдельно в опциях меню «Импорт», доступ к данной операции имеет только администратор системы. Эта функция предназначена для автоматического импорта CSV-файлов, которые автоматически экспортируются другими устройствами или системами. Различные устройства системы могут быть сконфигурированы на создание CSV-файлов по определенному графику и на копирование этих файлов в назначенную папку.

Эта функция полезна для тех тем которые постоянно генерируют новые данные, которые должны оперативно поступать в систему в целях отслеживания ситуации. Это могут быть, например, входные данные для создания запланированных отчётов или расчётов оценки геотехнических рисков. Типичным примером являются данные микросейсмических событий.

После конфигурации системы участие пользователя в процессах контроля качества данных, которые импортируются в базу данных и интегрируются в набор данных, не будет необходимо. Если в ходе этого процесса произойдет ошибка, системному администратору или другому назначенному пользователю будет направлено извещение по электронной почте.

Данная функция позволяет сократить количество ошибок, например, в результате манипулирования данными, поскольку она не требует вмешательства пользователя. Кроме того, импорт данных выполняется оперативно, независимо от графика работы пользователя.

Добавление или обновление контрольного файла:

  1. В меню Geoscience INTEGRATOR, нажмите на пункт Контролируемые папки в меню Импорт.
  2. Нажмите Добавить.
  3. В диалоговом окне Обновить контрольный файл выделите набор данных, шаблон, и имя группы, по которым будет выполняться контроль, затем добавьте любые необходимые метки и адрес электронной почты для включения в процесс контроля.
  4. Нажмите Обновить.

 

Импорт из других баз данных

Система предусматривает возможность создания настраиваемых соединений для прямого получения данных от источников данных других производственных объектов. Такие соединения создаются в рамках отдельных проектов, аналогично использованию контролируемых папок, преимущество таких методов заключается в отсутствии взаимодействия с пользователем, благодаря чему ограничивается зависимость от человеческого фактора.
