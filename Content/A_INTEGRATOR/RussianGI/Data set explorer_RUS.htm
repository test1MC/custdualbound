Диспетчер базы данных

Диспетчер базы данных является начальной точкой для просмотра наборов данных в отдельной теме для оценки и анализа содержащихся в них данных. Инструмент отображает список имен наборов данных, их описания и общую информацию о теме, что позволяет выбрать интересующие пользователя наборы. Также диспетчер предоставляет доступ к многочисленным ресурсам и инструментам, предназначенным для анализа данных и построения элементов отчетов, таких, как графики и таблицы.

Чтобы открыть Диспетчер базы данных:

  1. В меню Geoscience INTEGRATOR, нажмите на Диспетчер базы данных, при этом откроется список наборов данных, организованных в поле Активная тема.


  2. В таблице Выбрать набор данных, найдите и нажмите имя набора данных, который необходимо открыть.

 

В зависимости от масштаба проекта, в теме могут отсутствовать наборы данных, или может находиться один или несколько наборов данных или заполнителей наборов данных. При наличии наборов данных, в таблице отображается список их имен, их описания и общая информация о теме.

В зависимости от темы, в сводках наборов данных могут быть представлены максимальные и минимальные значения по определенным ключевым данным, а также могут быть представлены несколько информационных полей. Эти поля работают в роли каталогов, при нажатии на них, поля разворачиваются и отображают содержимое непосредственно в данном месте.

Сводки данных всегда содержат такие поля как: «Участки», «Файлы», «Метки», «Свойства». В зависимости от текущей темы, в Диспетчере базы данных могут находиться другие столбцы, например «Пробы», «Приборы», или «Датчики». Эти столбцы предназначены для предоставления пользователю дополнительной информации перед выбором набора данных и связаны контекстом с типом данных.

Диспетчер базы данных отдельно выводит список заполнителей наборов данных во второй таблице, под списком наборов данных с данными. Заполнители представляют собой наборы данных, под именами которых еще не выполнен импорт значений в базу данных. Им может быть присвоено имя и описание. Также с ними могут быть связаны файлы и метки. В ситуации, где ожидается поступление данных, но данные еще не поступили, заполнитель наборов данных может быть использован для подготовки набора данных перед импортом. Или, например, заполнитель может применяться, если в системе находятся данные в формате, который не поддерживается, тем не менее, необходимо предоставить пользователям доступ к этим данным и организовать их в системе под определённой темой.

 

Участки

Участки задаются на уровне проекта. Они представляют собой пространственные объемы, которые пользователи описывают по названию, а не по координатам. Так, например, в руднике участок может представлять собой горизонт, выемочную камеру, штрек или околоствольный двор. Участки и их границы указаны в меню Параметры проекта .

При разворачивании списка Участков для отдельного набора данных пользователь видит, в каких из Участков находится хотя бы одна из записей этого набора данных. Если участки не перечислены, это не означает отсутствия данных; это означает, что на заданные в проекте участки не приходится ни одной записи данных из рассматриваемого набора данных.

 

Файлы

При разворачивании списка файлов пользователь может видеть и загружать в систему все файлы, связанные с набором данных, или хотя бы одной записью из этого набора данных. Особое преимущество этой функции заключается в том, что Geoscience INTEGRATOR не только импортирует и сохраняет значения данных, но может также привязывать к набору данных или отдельным записям любое количество файлов.

Метки

Метки являются идентификаторами в виде текстовой строки, которые пользователи могут присваивать данным в системе Geoscience INTEGRATOR. При разворачивании списка меток для набора данных система отображает все метки, связанные с этим набором данных, или хотя бы одной записью из этого набора данных. При нажатии на метку в этом списке выполняется переход на страницу «Метки», где отображаются все данные, привязанные к этой метке.

Свойства

При разворачивании списка свойств на экране отображаются все свойства и их единицы, данные по которым присутствуют в наборе данных.
