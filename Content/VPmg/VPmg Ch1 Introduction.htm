Overview

VPmg is 3D modelling and inversion software for gravity, gravity gradient, total magnetic intensity (TMI), and magnetic gradient and amplitude data that utilizes the VP (vertical prisms) model. The program excels as a platform for creating geologically-constrained potential field inversions, while explicitly honouring drillhole information.

VPmg is part of VP Suite—a collection of geophysical modelling and inversion products. The discretization of the earth (required for all programs in the genre) sets the VP Suite apart from other available programs. The VP model directly links the rock properties and geologic units to the geophysics. The earth is laterally divided equally into vertical prisms (VP), which are then sub-divided in-depth to conform with undulating topographic and geologic layers.

Each VP model classifies geology units based on its average physical property, which we call homogeneous units. These units can be further divided into smaller cells to create heterogeneities within the unit. Heterogeneous cells may vary yet stay within the user-specified bounds of its geological unit.

This unique structure allows you to explore the geologic possibilities that reproduce the geophysical data, by performing actions such as:

  • Adjusting homogeneous units
  • Moving geologic horizons (geometry inversion)
  • Solving for heterogeneities (heterogeneous inversion) within a unit or units
  • Finding a variable basement under the units of interest

Every VP model has a VP basement. It is important to note that this may be the interpreted geologic "basement". This bottom unit is important because it will continue to the absolute bottom of the model (usually -25 km in elevation). The VP basement stems from the idea that the VP Suite modelling space is implicitly within the earth (see illustration below). This methodology allows you, for example, to recover density instead of density contrast.

VPmg is run through the command prompt as a standalone executable. A command-line utility (VPutility) is provided to aid in building and constraining VP models. VPmg underlying geological parameterization is capitalized upon when used in conjunction with geological modelling software, such as GOCAD® Mining Suite.

Scope

This guide will help you do the following:

  • Become proficient with VPmg to implement it to address your geophysical modelling and inversion needs.
  • Gain an understanding of VPmg file formats and features that you can use when confronting everyday geophysical problems.

Philosophy

Geophysical inversion is an iterative process: the result should be geologically reasonable and should include as much known information as possible. This drives our philosophy behind VP Suite by fully integrating geology at all phases of geophysical inversion. In general, VP Suite is designed to allow for hypothesis testing (e.g., exploratory modelling) and harmonizing geologic and geophysical data. We emphasize a hands-on iterative approach that validates the model and assumptions at each step. Yet, if nothing is known about the subsurface, an unconstrained inversion is completely reasonable. VPmg ultimately has the tools to hypothesis-test almost every kind of geologic scenario imaginable. The main advantage of using multiple styles of inversion is to enhance the flexibility and control of the process. Below are a few of the common problems VPmg addresses:

  • Terrain correction modelling
  • System-based modelling and inversion of gravity gradiometry data
  • Incorporating self-demagnetization
  • Dealing with remanent magnetization direction (solving / removing)
  • Depth-to-basement modelling
  • Processing through apparent susceptibility and density models (equivalent sources)
  • Unconstrained inversion
  • Handling regional effects within the area of interest
  • Creating a 3D lithologic body through inversion
  • Surface and borehole modelling and inversion

Updates from previous version

In version , we have further addressed bug fixes that were found. Upgrades and new features have also been introduced. A summary is of changes is:

  • Introduction of a rotated VP model
    • Model defined by an origin (e0, n0), side lengths, and rotation angle
    • Incision is allowed for models that are rotated at the same angle
  • RMI checks for Koenigsberger ratio or effective susceptibility. The inversion is output in given units (units can not be mixed)
  • Geosoft XYZ files (.xyz) can now be read as data files
    • Line information (if given) is output to end of model
    • Lines can be alphanumeric
    • Parameter file allows the line column with LINE keyword
  • No longer a maximum number of geologic units within a model file
  • Self-demagnetization now uses previous result for initialization of the optimization process
  • Number of RMS decimal places is increased depending on given uncertainty
  • Further robustness has been added to the half-space optimization for susceptibility
  • Updated initialization of susceptibility models for TMG and amplitude inversion. Added forward model after the first step, creating a more robust inversion result.
  • Fixed minor bugs in incision process:
    • partial prism computation in the south-west corner
    • fixed check for different VP model floor (elmin) values
  • Linear solver updated to a 1-step conjugate gradient
