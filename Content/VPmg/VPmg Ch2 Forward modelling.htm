Forward modelling

Once you have a VP model and a set of data locations, you can begin to forward model data responses. An optimal DC shift of the predicted data versus the observed data can be calculated during the forward modelling computation. A root-mean-squared (RMS) misfit of the given observed data compared to the predicted data is also an output. Both values aid in hypothesis testing on real data by giving you quantitative information on how well the computed data reproduce the observed data. You can model and invert the following data types:

  • Total magnetic intensity (TMI)
  • Magnetic components (Bx, By, Bz)
  • Magnetic amplitude data
  • Total magnetic gradient (analytic signal)
  • Magnetic tensors (all six components)
  • Magnetic tensor amplitude
  • Vertical gravity
  • FTG gradiometry (All six components)
  • Falcon gradiometry (GUV, GNE)
  • VK1 gradiometry (GGA, GGC)

The format of the control file for VPmg does not change from forward modelling to inversion. To forward model data, set up the control file and set the number of iterations to 0 (regardless of style of inversion).

VPmg writes the output file in the style of the VP model and places the computed data at the end of the file. This is useful to use as an initial model for a subsequent inversion. In this section you will find the observed, predicted, and background data described. Details of how the background data are computed are found in the Regional and local models section.

_amp, placed before the file extension. For example, VPmg will create
myData_amp.dat if myData.dat was the data file.
